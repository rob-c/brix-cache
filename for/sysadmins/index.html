For grid sysadmins

Add TLS, auth, and metrics. Change nothing downstream.

BriX-Cache is an nginx module — so your storage endpoint inherits nginx's worker model, graceful reloads, and operational muscle memory. Deploy it standalone, or slide it in front of what you already run.

in front of an existing server
# stream
brix_root on;
brix_tap_proxy on;
brix_tap_proxy_upstream ceph-xrootd:1094;
# clients connect to nginx — backend stays invisible
What you get on day one

Operationally boring, on purpose.

drop-in

Front any XRootD service

Run in transparent-proxy mode and BriX-Cache terminates TLS and auth, emits metrics, then relays opcodes verbatim to your backend — lazy upstream connect, end-to-end file-handle translation, backend identity never exposed. No client or backend config changes.

modes

Three deployments, one instance

Standalone server over a POSIX tree, transparent root:// proxy, or WebDAV perimeter proxy that terminates HTTPS + WLCG tokens and forwards inward. The stream block owns native traffic, http owns WebDAV / S3 / metrics. Mix them freely.

fail-early

Bad config fails at nginx -t

Missing certificates, JWKS files, CRLs, or required directories make nginx -t fail with an explicit emerg line — before any traffic is accepted. Reloads drain gracefully: in-flight transfers finish on old workers while new connections pick up the new config.

observe

Prometheus + a live dashboard

Per-request counters for every protocol with fixed, low-cardinality labels, structured access logs, and an HTTPS operator dashboard at /brix/ showing active transfers, cache and cluster health, and versioned JSON under /brix/api/v1/.

cache

Read-through cache built in

XCache-style direct-mode fills from an anonymous root:// / roots:// origin with per-file worker locks, with optional write-through mirroring on kXR_sync / kXR_close.

cluster

Manager & clustering

CMS heartbeat, a dynamic server registry, kXR_redirect / kXR_locate, and an S3 gateway — the pieces for a multi-node storage federation, assembled from the same module.
