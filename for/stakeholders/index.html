Why it's a safe bet

Credibility from what it's made of.

wlcg

WLCG-native by design

It speaks the protocols and identity systems the grid actually mandates — XRootD, WebDAV, GSI / x509 with VOMS, and WLCG bearer tokens with scoped access — the same machinery used to move petabytes of collision data at labs like CERN, SLAC, and Fermilab.

foundation

Built on nginx, not from scratch

The hard, security-critical parts — the event loop, TLS, connection handling, process model, and graceful reloads — are nginx's, hardened over two decades in production. BriX-Cache adds the protocol layer on top, so the foundation isn't a research prototype.

proven

Proven by conformance testing

A comprehensive Python suite runs the same tests against BriX-Cache and the reference xrootd daemon — native root://, the XrdHttp WebDAV endpoint, auth, ACLs, proxy and manager modes. Divergence is treated as a defect.

resilient

Resilient where it counts

On a healthy network it matches reference-XRootD throughput; on a degraded one, transfers still finish byte-exact instead of failing — the difference between a lost dataset and a slow one.

open

Open source, no vendor lock-in

Released under AGPL-3.0-only: inspect it, modify it, run it yourself. If you deploy a modified version over a network, you make your source available to its users — the reciprocity that keeps shared infrastructure shared.

consolidate

One service, three interfaces

Grid, cloud, and web access to the same storage from a single deployed component — less software to run, patch, and staff than three separate gateways doing the same job.

One storage service the grid already understands.
