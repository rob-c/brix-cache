4/*
5 * site_n2n.h — pluggable, tunable site name-translation (LFN ↔ physical name).
6 *
7 * WHAT: A pure (libc-only) translation from a logical path (the wire LFN, export-
8 * relative, leading slash) to the PHYSICAL name a backend addresses, plus
9 * the reverse for directory listing. It models the real GridPP Ceph site
10 * conventions so one driver serves many sites by config alone.
11 *
12 * WHY: "ceph.namelib" means different things per architecture (see the design
13 * spec §1.0a). RAL/Glasgow (XrdCeph/RADOS) name objects "<pool>:<prefix><lfn>"
14 * — the pool is a ':'-prefix that stock XrdCephOss::extractPool splits off.
15 * CephFS sites (Lancaster/Manchester/Brunel) are POSIX, so the physical name
16 * is a path "<localroot><lfn>". This module is the single, tunable home for
17 * both, so the rados driver and a posix-on-CephFS export both reach storage
18 * the way the site already wrote its Pb+ of data.
19 *
20 * HOW: Scheme + a few string knobs (pool, prefix/localroot). Every translation
21 * rejects path traversal ("..") in the LFN. Dependency-free (unit-tested
22 * standalone; links into the module and libxrdproto).
23 */
24
