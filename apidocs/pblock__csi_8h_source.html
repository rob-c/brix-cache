4/*
5 * pblock_csi.h — F3 per-block CRC32c integrity (CSI) for the pblock driver.
6 *
7 * WHAT: One CRC-32c (Castagnoli, INVARIANT 1/9) per pblock block, at rest in a
8 * `csi(blob_id, block_no, crc)` catalog table. Keyed by blob_id — NOT
9 * logical path — so a rename (which keeps the blob) never re-tags. The
10 * driver verifies fully-spanned blocks on read (mismatch ⇒ EIO, never
11 * serve garbage) and flushes recomputed CRCs on close.
12 *
13 * WHY: pblock is the only full-parity driver missing filesystem page checksums
14 * (BRIX_SD_CAP_FSCS). The posix CSI tagstore (csi_tagstore.h) carries tags
15 * in an xmeta record at a real POSIX path — meaningless for pblock, whose
16 * data is sharded blobs with nothing at the logical path. So pblock owns
17 * its integrity in its own store (the catalog), advertised as CAP_FSCS
18 * only when csi=1 (honest per-instance capability).
19 *
20 * HOW: Opt-in `csi=1` (rides the `?tail` static-opts channel like audit/lab).
21 * The at-rest CRC row set for the open object is snapshotted into a
22 * handle-local array ONCE at open — the hot read path verifies against
23 * memory with zero DB I/O and zero locks, matching the pblock prime
24 * directive. Writes only widen an integer [dlo,dhi) block extent on the
25 * hot path; close reads those blocks back and UPSERTs their CRCs in one
26 * transaction. ngx-free (libc + sqlite3 + the crc32c helper), gated by
27 * BRIX_HAVE_SQLITE like the rest of the backend.
28 *
29 * Requires: sd_pblock_catalog.h (pblock_catalog), pblock_store.h
30 * (pblock_state_t), <stdint.h>, <sys/types.h> before inclusion.
31 */
