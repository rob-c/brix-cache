4/*
5 * src/protocols/common — cross-protocol common code and the protocol-descriptor
6 * registry (QUALITY_ROADMAP 2.1: "common Protocol Handler Interface").
7 *
8 * nginx hosts two module families with fundamentally different request models,
9 * so the roadmap's generic open/read/write/close vtable does not fit: root:// is
10 * a STREAM module (an event-driven opcode state machine) while s3 / webdav /
11 * cvmfs are HTTP modules (method dispatch through nginx's phase handlers). The
12 * descriptor therefore captures what every protocol genuinely shares — a stable
13 * wire name and its module family — and is the single seam the dispatcher and
14 * observability layers can key off without inventing a false uniform vtable.
15 * (Protocol-specific request handling stays in each protocol's own module; the
16 * unified proto id/label table remains src/core/types/proto_list.h.)
17 */
