21/*
22 * ngx_http_brix_shared_conf_t — Common fields embedded at the top of every
23 * protocol location/server config struct (stream, WebDAV, S3).
24 *
25 * WHAT: A shared preamble that holds enable flags, root path, write permission,
26 * and thread pool name — fields present in all three protocol configs. Each
27 * protocol struct embeds this struct as its first member so offsetof() offsets
28 * into the protocol-specific tail remain valid after merge.
29 *
30 * WHY: Stream, WebDAV, and S3 each duplicate enable + root + allow_write in
31 * their own structs and their create/merge functions (~90 total ngx_conf_merge_*
32 * calls). Consolidating these shared fields into one struct reduces merge
33 * boilerplate to ~30 protocol-specific calls plus a single preamble merge.
34 *
35 * HOW: Protocol structs declare this as their first member (no padding needed
36 * because it starts with ngx_flag_t which aligns naturally). The create function
37 * sets all shared fields to NGX_CONF_UNSET; the merge function uses standard
38 * nginx merge macros on each field before calling protocol-specific merge logic.
39 */
40
