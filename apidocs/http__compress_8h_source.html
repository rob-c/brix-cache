1/*
2 * http_compress.h — outbound (GET) HTTP response compression.
3 *
4 * WHAT: Accept-Encoding negotiation + on-the-fly response compression for the
5 * shared file-serve path (WebDAV GET / S3 GetObject). Off by default
6 * (per-location config); when enabled and the client advertises a codec we
7 * support, the GET body is streamed compressed with a Content-Encoding +
8 * Vary header instead of via zero-copy sendfile.
9 * WHY: phase-42 outbound parity/extension — XRootD's HTTP stack does no content
10 * compression; this lets clients fetch compressible objects pre-compressed.
11 * HOW: negotiate() picks the best available codec from Accept-Encoding (skipping
12 * ranges, tiny files, and already-compressed MIME types); send_compressed()
13 * sets chunked headers and streams read -> codec compress -> output filter.
14 */
