1/*
2 * stream.c — Prometheus exporter for native XRootD (stream-layer) counters.
3 *
4 * WHAT: Maps numeric operation slots to human-readable label strings
5 * (`brix_op_names[]`) and iterates the shared-memory metrics zone
6 * to emit Prometheus exposition-format lines for all stream-protocol
7 * counters — connections, bytes, wire frames, request/reject stats,
8 * per-operation ok/error counts, path-depth violations, registry full.
9 *
10 * WHY: The stream module writes counters into `ngx_brix_metrics_t` shared
11 * memory using atomic fields. This file is the HTTP-side exporter that
12 * reads those counters and formats them as Prometheus scrape output.
13 * The op-name mapping is a binary ABI between stream and HTTP modules:
14 * slot indices must stay aligned with `BRIX_OP_*` constants in metrics.h.
15 *
16 * HOW: `brix_export_prometheus_metrics()` iterates all server slots, reads
17 * each atomic counter via `ngx_atomic_fetch_add(..., 0)` for an
18 * eventually-consistent snapshot, and writes HELP/TYPE/value lines via
19 * the `metrics_writer_t` interface. Uses macro templates to reduce
20 * repetition. Calls protocol-specific exporters (webdav, s3, proxy,
21 * tracking) at the end.
22 */
23
