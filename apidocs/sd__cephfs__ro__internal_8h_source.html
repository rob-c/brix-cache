1/*
2 * sd_cephfs_ro_internal.h — private declarations shared across the read-only
3 * CephFS-via-RADOS driver's translation units.
4 *
5 * WHAT: The "cephfsro" driver is split across three .c files for file size and
6 * auditability — sd_cephfs_ro.c (lifecycle + open/read/stat/xattr + the
7 * driver descriptor), sd_cephfs_ro_resolve.c (the metadata-decode +
8 * inode-resolution path) and sd_cephfs_ro_dir.c (directory iteration). This
9 * header carries only the handful of symbols one file DEFINES and another
10 * REFERENCES: the per-export driver state, the retry/resolve helpers the
11 * data + directory paths call, and the directory vtable ops the descriptor
12 * wires up. Everything used within a single file stays static there.
13 *
14 * WHY: Keeping cross-TU coupling to one small header makes the split auditable —
15 * no symbol leaks except the ones listed here.
16 *
17 * This header is only ever included from within a `#if BRIX_HAVE_CEPH` region
18 * (the whole driver is an empty TU otherwise); it references librados-gated types
19 * (sd_ceph_conn_t, brix_sd_* ) that exist only under that guard.
20 */

27/* per-export: connections to the metadata + data pools, plus the consistency
28 * mode. `live` enables best-effort tracking of a still-mounted fs: optimistic
29 * walk-version revalidation (detect an MDS write that landed mid-walk and retry)
30 * and a bounded retry of a not-found whose path raced. `quiesced` mode trusts the
31 * fs is frozen and only retries genuinely transient cluster errors. */
