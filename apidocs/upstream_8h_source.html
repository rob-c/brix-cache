6/*
7 * Upstream redirector query subsystem.
8 *
9 * When brix_manager_mode is enabled and a kXR_locate or kXR_open finds a
10 * server in the registry, the server does NOT redirect immediately. Instead it
11 * uses the upstream module to make a lightweight XRootD "locate" query to the
12 * registered data server, confirming the file exists, then replies with the
13 * data server address as a kXR_redirect response.
14 *
15 * State machine integration:
16 * brix_upstream_start() is called from the opcode handler.
17 * ctx->state = XRD_ST_UPSTREAM is set; the client-facing read event is disarmed.
18 * When the upstream response arrives, ctx->state is restored and the
19 * appropriate kXR_redirect or kXR_error response is sent to the client.
20 * brix_upstream_cleanup() is called from brix_on_disconnect() to release
21 * all upstream resources even if the upstream query is still in flight.
22 */
23
24/*
25 * brix_upstream_start — initiate a TCP connection to the selected upstream
26 * data server, post the locate query, and wait for the response.
27 *
28 * Returns NGX_OK (ctx->state = XRD_ST_UPSTREAM on success, error response
29 * already queued on NGX_ERROR).
30 */
