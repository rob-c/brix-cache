9 * WHAT: Decodes a percent-encoded URI string (e.g. `/data/my%20file.root`) into a
10 * plain-text destination buffer. Rejects embedded null bytes per HTTP semantics.
11 *
12 * WHY: WebDAV clients send URIs in percent-encoded form. Null bytes in URIs indicate
13 * malformed input and must produce a 400 response rather than truncating the
14 * decoded string (which would cause incorrect path matching).
15 *
16 * HOW: Wraps `brix_http_urldecode()` from src/compat/uri.h with nginx status-code

20 * RETURN: NGX_OK on success; nginx error code on failure (never raw compat codes).
