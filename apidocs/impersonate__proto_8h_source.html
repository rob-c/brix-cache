4/*
5 * impersonate_proto.h — worker <-> broker wire protocol (phase 40).
6 *
7 * WHAT: The fixed-size request/reply frames the unprivileged worker exchanges
8 * with the privileged root broker over an AF_UNIX socket. The worker sends a
9 * request naming the authenticated principal + a root-relative path + the op;
10 * the broker maps the principal to UNIX creds, performs the op under those
11 * creds with RESOLVE_BENEATH confinement, and replies — passing an open file
12 * descriptor back via SCM_RIGHTS for OPEN.
13 *
14 * WHY: A fixed-size frame (like the FRM stage-agent protocol) keeps framing
15 * trivial and bounded — no parser, no partial-frame state machine beyond a
16 * read-full loop. Paths are root-relative (the broker holds the authoritative
17 * rootfd), so the worker can never name an absolute path or escape the export.
18 *
19 * HOW: One frame type each way. `imp_req_t` carries op + flags + mode + length
20 * + bounded principal/path/path2 buffers. `imp_rep_t` carries a status
21 * (0 = ok, negative = -errno, positive = an IMP_STATUS_*), reply flags, and a
22 * portable stat subset. Both are POD; the broker and the worker are the same
23 * architecture (a local socket), so no byte-swapping.
24 */

72/* Reply status (in imp_rep_t.status). 0 = success. Negative = -errno from the
