22/* Confined extended-attribute ops — broker-routed under impersonation.
23 *
24 * WHAT: set/get/remove/list extended attributes on a resolved (already
25 * lexically-confined) path. When impersonation map mode is active the op
26 * is routed to the privileged broker, which re-confines under its own
27 * export rootfd (openat2 RESOLVE_BENEATH) and performs the f*xattr as the
28 * mapped user — so the lock/dead-property xattr lands with, and is
29 * DAC-checked for, the real user (not the unprivileged worker). When
30 * impersonation is off the behaviour is byte-for-byte the prior raw
31 * path-based syscall, so the non-impersonated path is unchanged.
32 *
33 * WHY: WebDAV LOCK tokens and PROPPATCH dead-properties are stored as `user.*`
34 * xattrs on the resource. Without broker routing the worker (svc) would
35 * attempt setxattr on a file owned 0644 by the mapped user and fail EACCES
36 * — i.e. LOCK/PROPPATCH were broken under impersonation. Routing fixes
37 * that and keeps the on-disk metadata owned by the right identity.
38 *
39 * Return values mirror the POSIX *xattr contract (get/list: byte count, or the
40 * size when bufsz==0; -1/ERANGE when the caller buffer is too small).
41 */
