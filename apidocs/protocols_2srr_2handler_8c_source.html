1/*
2 * handler.c — HTTP content handler for the WLCG SRR endpoint.
3 *
4 * WHAT: ngx_http_brix_srr_handler() serves the storageservice JSON document
5 * for a location carrying `brix_srr on;`. GET/HEAD only; request body is
6 * discarded; no request input influences the output.
7 *
8 * WHY: Mirrors the /metrics handler (src/metrics/handler.c) so the two
9 * read-only reporting endpoints behave identically (method gating, body
10 * discard, NGX_DECLINED when disabled so the location can be toggled without
11 * a 404). The payload is sized to the document via the builder's two-pass
12 * json_dumpb().
13 *
14 * HOW: build → set status/headers (application/json) → send_header → output the
15 * single memory buffer. HEAD stops after the header.
16 */
