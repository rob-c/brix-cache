4/* ---- Write-through decision interface — policy evaluation at kXR_open ----
5 *
6 * WHAT: Interface for evaluating write-through policy when a file is opened.
7 * Decision cached on handle determines close-time flush behavior (sync/async/deny).
8 *
9 * WHY: Mirrors XrdPfcDecision::Decide() from official XRootD PFC module but implemented as C callbacks.
10 * Function-pointer pattern allows external plugins to provide custom policy engines without recompiling.
11 *
12 * HOW: Decision flow mirrors XrdPfc::Cache::Decide():
13 * 1. Client opens file with kXR_open options indicating write intent
14 * 2. At open time, evaluate brix_wt_decision_fn(path, options, user_data) — default is brix_wt_default_decide()
15 * 3. Cache result on handle (brix_file_t): wt_policy, wt_enabled, wt_dirty_offset = -1
16 * 4. Close-time: a real write-back implementation can use the cached policy
17 * and dirty-range metadata to decide whether and how to flush.
18 *
19 * INVARIANT: Decision is evaluated ONCE at open time — never re-evaluated for each write. */
20
