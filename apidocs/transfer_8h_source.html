4/* ---- Module: TPC Transfer Vocabulary ----
5 *
6 * WHAT: The shared vocabulary for third-party copies — protocol tags
7 * (STREAM / WEBDAV), direction tags (PULL / PUSH), lifecycle states
8 * (PENDING / ACTIVE / DONE / ERROR), the registry sizing constants
9 * (slot count and max URL/path lengths), and the protocol-neutral
10 * brix_tpc_transfer_t describing one in-flight copy.
11 *
12 * WHY: Every other file in tpc/common builds on these definitions, so they live
13 * in one leaf header with no dependencies beyond nginx core. Keeping the
14 * state machine and size limits centralised guarantees the stream and
15 * WebDAV transports, the registry, and the metrics/dashboard readers all
16 * agree on the same enum values and buffer bounds.
17 *
18 * HOW: Plain #define tags and a POD struct. The brix_tpc_transfer_t ngx_str_t
19 * members (src_url, dst_path) point at storage owned by the registry slot
20 * that holds the transfer; callers may pass stack/request-pool strings to
21 * brix_tpc_registry_add(), which copies them before publishing.
22 */
