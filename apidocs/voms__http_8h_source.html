4/*
5 * voms_http.h — lightweight VOMS extraction declaration for HTTP handlers.
6 *
7 * WHAT: Declares brix_extract_voms_info() without pulling in the full
8 * stream-module umbrella header (ngx_brix_module.h), which drags in
9 * ngx_stream.h and stream-specific types incompatible with HTTP modules.
10 *
11 * WHY: WebDAV auth_cert.c needs to call VOMS extraction after X.509 proxy
12 * chain verification, but including ngx_brix_module.h from HTTP code
13 * causes type redefinition conflicts with nginx's HTTP layer.
14 *
15 * HOW: This thin header re-declares only brix_extract_voms_info() using
16 * types already available in every HTTP module (ngx_core.h + OpenSSL).
17 * The implementation lives in src/voms/extract.c, which is compiled into
18 * NGX_ADDON_SRCS and linked into the module regardless of which header
19 * declares the prototype.
20 *
21 * Usage:
22 * - Include this header in HTTP handler code that calls VOMS extraction.
23 * - Do NOT include ngx_brix_module.h alongside this header — they both
24 * declare brix_extract_voms_info() and will conflict.
25 */
