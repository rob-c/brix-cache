1/*
2 * xfer_spawn.h — crash-safe synchronous external-command runner.
3 *
4 * WHAT: brix_xfer_run_reparented() runs an argv (no shell) in a double-forked,
5 * reparented-to-init agent so the calling process manager (nginx) never
6 * reaps it, blocks until it finishes, and returns the command's exit status.
7 *
8 * WHY: Some transfers shell out to an external client (write-through's GSI origin
9 * upload, and — later — TPC). Those used posix_spawn + waitpid, which leaves
10 * a direct child of the nginx worker; reaping a worker child can crash the
11 * master via the SHM-zone-as-slab-pool SIGCHLD handler (see the FRM fork
12 * postmortem). This is the synchronous sibling of the long-lived agent
13 * harness (xfer_mover_agent.c): same reparent-so-nginx-never-reaps guarantee,
14 * but call-and-block-for-exit-code, which is what those upload paths need.
15 *
16 * HOW: Deliberately ngx-free (POSIX only) so it is unit-testable standalone
17 * (src/fs/xfer/xfer_spawn_unittest.c) and callable from any context — the
18 * event loop or an nginx thread-pool worker. The child only performs
19 * async-signal-safe operations between fork and execve, so it is safe to
20 * call from a multithreaded (thread-pool) context.
21 */
22
