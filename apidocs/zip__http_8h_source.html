1/*
2 * zip_http.h — shared HTTP ZIP-member serving (phase-57 W2), used by WebDAV GET
3 * and S3 GetObject.
4 *
5 * WHAT: Extract a "?xrdcl.unzip=<member>" query arg and serve that member of a
6 * ZIP archive over an HTTP response (stored via sendfile byte-range,
7 * deflate via in-RAM inflate, with single-Range support).
8 * WHY: HTTP/S3 clients cannot self-inflate a ZIP member (unlike root:// XrdCl),
9 * so the server must extract. WebDAV and S3 share one implementation.
10 * HOW: Confined archive open + brix_zip_find_member (zip_dir.h) +
11 * brix_zip_extract_full for deflate; a dup'd fd kept alive by an
12 * ngx_pool_cleanup_file for the stored sendfile path.
13 */
