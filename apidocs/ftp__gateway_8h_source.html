4/*
5 * gridftp/ftp_gateway.h — GridFTP / FTP control-channel gateway (phase-82 P82.1).
6 *
7 * A self-contained nginx STREAM module that terminates an RFC 959 FTP control
8 * channel and bridges STOR/RETR/LIST/SIZE/MKD/DELE to the VFS storage seam
9 * (brix_vfs_*), confining every path through brix_http_resolve_path(). This is
10 * the cleartext vertical slice: no RFC 2228 GSI security layer yet (AUTH GSSAPI
11 * is advertised absent), PASV-only passive data channel, TYPE I transfers.
12 *
13 * Directive: brix_gridftp on; (inside a stream server {} block)
14 * Required: brix_gridftp_export <dir>; (the exported filesystem tree root)
15 * Optional: brix_gridftp_allow_write on; (permit STOR/MKD/DELE; default off)
16 *
17 * Scope note (POC): the control + passive-data dialogue runs synchronously on a
18 * blocking socket inside the worker for a single client at a time. This proves
19 * the VFS round-trip end to end; the production form is the event-driven state
20 * machine described in docs/refactor/phase-82-gridftp-gateway.md §5.
21 */
