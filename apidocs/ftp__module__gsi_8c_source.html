30/* brix_ftp_build_gsi — construct the host TLS context (cert/key) and the client
31 * proxy trust store once the GSI directives are known. Unlike root:// / WebDAV
32 * TLS, the mem-BIO GSSAPI engine (gsi_mech.c) drives handshakes on a bare SSL
33 * object with no nginx connection attached, so we must NOT use ngx_ssl_create():
34 * it installs nginx info/servername callbacks that deref an ngx_connection_t via
35 * SSL ex-data our SSL never has, crashing mid-handshake. A plain OpenSSL
36 * SSL_CTX (as in the phase-82 interop probe) sidesteps every such callback. */
