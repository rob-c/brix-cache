1/*
2 * metabench.h — deterministic metadata-storm workload generator + latency
3 * percentiles, shared by tests/tools/pblock_meta_bench.c and
4 * client/apps/diag_metabench.c. Private client header (not a public API).
5 *
6 * WHAT: one worker owns subtree /w<id>/d<d>/f<f>; the program is fixed so the
7 * expected post-create namespace is known exactly — catalog integrity is a
8 * strict equality check, not a smoke test.
9 * WHY: prove pblock metadata correctness + performance under concurrency.
10 * HOW: this header/TU is PURE (planning + percentile + result printers, no wire
11 * I/O) so it compiles + unit-tests standalone with only libc. The threaded
12 * GSI runner that needs libbrix lives in metabench_run.{c,h}.
13 */
