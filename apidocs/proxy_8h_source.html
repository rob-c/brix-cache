4/*
5 * proxy.h — public API for the XRootD transparent proxy module.
6 *
7 * When brix_proxy is enabled for a server block, every post-login opcode is
8 * forwarded to the configured upstream XRootD server. The proxy:
9 *
10 * 1. Authenticates the client normally (token, GSI, sss, or anonymous).
11 * 2. Lazily connects to the upstream on the first non-session opcode and
12 * completes a standard XRootD bootstrap (handshake + protocol + login).
13 * 3. Translates client-assigned file handles to upstream-assigned ones for
14 * open/read/write/close opcodes.
15 * 4. Relays upstream responses verbatim (or with fhandle rewriting) back to
16 * the client, preserving the client's streamid.
17 * 5. Collects per-request metrics and emits a JSON audit record at close.
18 *
19 * The upstream connection is unauthenticated (anonymous login) in Phase 1;
20 * auth bridging (GSI delegation, token forwarding) is a Phase 4 concern.
21 */
