4/*
5 * webdav.h — Shared types, configuration structs, and function declarations for the
6 * nginx HTTP WebDAV module (davs:// endpoint).
7 *
8 * WHAT: Declares all shared data structures used across WebDAV source files:
9 * location-level config (`ngx_http_brix_webdav_loc_conf_t`), per-request
10 * auth context (`ngx_http_brix_webdav_req_ctx_t`), lock table entry and
11 * shared-memory table, authentication enums, and constants. Also declares
12 * every public function across the WebDAV module — path resolution, URI
13 * utilities, XML escaping, metrics, authentication, HTTP method handlers,
14 * file I/O helpers, HTTP-TPC (third-party copy), upstream proxy, and
15 * credential delegation.
16 *
17 * WHY: This header is included by every webdav source file so that all modules see
18 * the same types in a single include. It also includes xrdhttp.h so that both
19 * WebDAV and XrdHttp types coexist — pointer casting between req_ctx_t and
20 * xrdhttp_req_ctx_t relies on layout compatibility (C11 §6.7.2.1p15). Without
21 * this unified header, each file would need to re-declare overlapping types.
22 *
23 * HOW: The header is organized into sections by responsibility:
24 * 1. Includes — nginx core + module headers, shared protocol/compat headers
25 * 2. Constants — path max, fd table size, lock sizes, TPC limits
26 * 3. Enums — auth modes (NONE/OPTIONAL/REQUIRED), proxy auth modes
27 * 4. LOCK structures — entry and shared-memory table
28 * 5. Location config struct — all directives mapped to fields with inline comments
29 * 6. Per-request context struct — auth results, token scopes, lock metadata
30 * 7. Module wiring — create/merge loc conf, postconfig, access handler, content handler
31 * 8. Operation registry — extern declarations for the operation capability table
32 * 9. Path/URI/XML/logging utilities — resolve_path, urldecode, escape_xml, cors
33 * 10. Authentication — ca store build, cert verification, bearer token validation
34 * 11. HTTP method handlers — options, head, get, put, delete, mkcol, propfind, move, copy, lock, unlock
35 * 12. File I/O helpers — fadvise, write_full, spooled file copy
36 * 13. HTTP-TPC — header macros, curl pull/push, thread task posting, COPY handler
37 * 14. Upstream proxy — handler, URL parsing
38 * 15. Credential delegation — mode parse, token obtain, metric names
39 * 16. Capability table macro — BRIX_WEBDAV_ALLOW_FLAGS()
40 *
41 * INCLUDES: xrdhttp.h is included here so all webdav source files see both types in
42 * a single include. See src/webdav/xrdhttp.h for the XrdHttp extension API.
43 */
