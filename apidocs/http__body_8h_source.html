10/* Default decompression-bomb expansion-ratio ceiling for inbound (PUT) decode of
11 * untrusted bodies: reject once output/input exceeds this (engages after 64 KiB
12 * output). 1000:1 is well past any legitimate payload but short of a zip-bomb. */
13#define BRIX_DECODE_MAX_RATIO 1000
14
15/* LZ4 frames do not reach the generic 1000:1 bomb threshold on zero-heavy
16 * payloads, so untrusted HTTP decode uses a codec-specific ceiling that still
17 * leaves ample room for ordinary compressible objects while rejecting compact
18 * LZ4 bombs before they are stored. */
19#define BRIX_DECODE_LZ4_MAX_RATIO 200
20
21/* Absolute output ceiling for inbound (PUT) decode, complementing the ratio
22 * guard. The ratio guard catches the classic tiny-input/huge-output bomb, but a
23 * body whose expansion stays just under the ratio (e.g. ~900:1) yet is uploaded
24 * near client_max_body_size could still decode to an enormous absolute size and
25 * exhaust disk. This finite per-request ceiling bounds that: 16 GiB is far above
26 * any realistic single compressed PUT object (large objects are streamed/multipart
27 * or stored uncompressed) yet caps the staged temp a hostile upload can produce.
28 * A tripped cap maps to 413, exactly like the ratio guard. */
29#define BRIX_DECODE_MAX_OUTPUT (16ULL * 1024 * 1024 * 1024)

160/*
161 * brix_http_body_inflate_to_fd - decompress and write request body to an fd.
162 *
163 * WHAT: Decompress the nginx request body chain using zlib inflate and write the
164 * decompressed bytes to dst_fd. Handles both memory-backed and file-backed
165 * (spooled) buffers in the nginx body chain.
166 *
167 * WHY: WebDAV PUT and S3 PUT clients may send Content-Encoding: gzip or deflate.

173 *
174 * Parameters:
175 * window_bits — 31 (15+16) for gzip, 15 for deflate; passed to inflateInit2
176 * summary_out — optional; filled with compressed body byte count if non-NULL
177 */

183 * brix_http_body_decode_to_fd - decompress a request body with any codec.
184 *
185 * WHAT: Streams the given codec (brix_codec_id_t — gzip/deflate/zstd/xz/brotli/
186 * bzip2) over the request body chain, writing plaintext to dst_fd, bounded
187 * by the decompression-bomb guard.
188 * WHY: WebDAV/S3 PUT may carry any supported Content-Encoding; the stored object

196 * max_output — hard output ceiling in bytes (0 = unbounded). Callers
197 * should pass a sane cap for untrusted uploads.
