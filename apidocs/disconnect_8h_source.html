5/*
6 * brix_on_disconnect — tear down all per-connection state when a client
7 * disconnects (read EOF, write error, or timeout).
8 *
9 * Responsibilities:
10 * 1. Sets ctx->destroyed = 1 so any in-flight AIO callbacks can detect the
11 * stale pointer and abort safely.
12 * 2. Cancels any pending upstream redirector query.
13 * 3. Frees heap-owned buffers (payload_buf, prepare_paths).
14 * 4. Frees OpenSSL state (gsi_dh_key, sigver MAC handle/context).
15 * 5. Decrements the active-connections metric and accumulates session I/O totals.
16 * 6. Emits access-log entries for any files still open at disconnect time
17 * (with "interrupted" status and throughput).
18 * 7. If the session was authenticated and not a bound secondary, removes the
19 * session from the shared registry (so kXR_bind cannot reuse this sessid).
20 * 8. Emits the DISCONNECT access-log entry with session-level throughput.
21 *
22 * NOTE: This function does NOT close open file handles — the caller must call
23 * brix_close_all_files(ctx) before or after this call.
24 */
