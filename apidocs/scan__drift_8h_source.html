1/*
2 * scan_drift.h — namespace↔catalog reconciliation set (D2 "object-store fsck").
3 *
4 * WHAT: a pure, ngx-free key→size set seeded from the backend's physical object
5 * catalog; the namespace walk is then streamed against it to classify each
6 * logical path (in-both / size-mismatch / namespace-only) and, afterward,
7 * to surface catalog keys never matched (orphan objects with no namespace
8 * entry).
9 * WHY: this is the algorithmic heart of `drift` — isolating it from the VFS /
10 * driver makes it unit-testable (scan_unittest.c) and keeps peak memory at
11 * O(catalog): the catalog side is held; the namespace side streams.
12 * HOW: open-addressing hash map (linear probe, power-of-two capacity); add()
13 * seeds the catalog, match() marks + classifies a namespace key, orphans()
14 * iterates the unmarked remainder.
15 */
