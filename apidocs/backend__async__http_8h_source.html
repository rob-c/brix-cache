1/*
2 * backend_async_http.h — http-plane (S3 / WebDAV) adapter for the durable async
3 * backend-op queue (src/fs/xfer/backend_async_queue.{c,h}).
4 *
5 * WHAT: brix_baq_http_try() enqueues a namespace mutation on the per-worker
6 * coalescing queue and PARKS the http request (r->main->count++, returns
7 * NGX_DONE) until the batch flushes; the supplied render callback then
8 * produces the real response and finalises the request on the event loop.
9 *
10 * WHY: The root:// plane parks via its recv state machine; the http plane has
11 * no such state, but nginx already models "hold the request, finish later"
12 * via r->main->count++ + a deferred finalise (see webdav/copy.c collection
13 * offload). This wraps that pattern so S3 and WebDAV share ONE park/resume
14 * path — the queue itself stays protocol-agnostic.
15 *
16 * HOW: The queue's flush fires as a posted event / 1s-tick on the event loop, so
17 * the wake (render + ngx_http_finalize_request) runs there too — no thread
18 * safety concern. The park record lives on r->pool, kept alive by the count
19 * reference until the render finalises. root_canon/src_key/dst_key are
20 * copied into the fsync'd durable record, so they need not outlive the call.
21 */
