4/*
5 * core/http/ktls.h — enable kernel-TLS (kTLS) on an nginx http_ssl context.
6 *
7 * WHAT: brix_http_ktls_enable_ctx() sets SSL_OP_ENABLE_KTLS on a server's
8 * already-built OpenSSL SSL_CTX so that HTTPS responses sendfile(2) over
9 * kernel-TLS (the kernel encrypts the records in place) and request bodies
10 * decrypt in-kernel — cutting the userspace SSL_write/memmove crypto that
11 * dominates the TLS data path.
12 *
13 * WHY: The HTTP plane (WebDAV, S3) serves files through nginx's native output
14 * chain, which already uses sendfile; once the SSL_CTX is kTLS-opted,
15 * ngx_linux_sendfile()/recv use kTLS automatically with NO handler change.
16 * This is the HTTP twin of the root:// stream path's SSL_OP_ENABLE_KTLS
17 * (src/protocols/root/session/tls_config.c).
18 *
19 * HOW: Runtime-safe. SSL_OP_ENABLE_KTLS is a transparent no-op when the
20 * negotiated cipher or the running kernel cannot offload — OpenSSL then
21 * falls back to userspace TLS. Each protocol calls this from its own
22 * postconfiguration server loop for the servers it owns (proper layering:
23 * this helper only touches the SSL_CTX it is handed).
24 */
