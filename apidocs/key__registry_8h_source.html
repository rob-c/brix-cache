6/*
7 * TPC key registry — shared-memory table of in-flight TPC keys.
8 *
9 * When the destination server accepts a write-open with tpc.src=, it
10 * generates (or echoes) the tpc.key and registers it here. The source
11 * server can then validate the key before serving the file. Keys expire
12 * after the configured TTL (default BRIX_TPC_KEY_TTL_MS; override with
13 * brix_tpc_key_ttl) to prevent resource leaks if
14 * the rendezvous never completes.
15 *
16 * The registry is per nginx worker group (shared memory). Rendezvous keys
17 * are registered and consumed on the *source* data server when the client
18 * opens with tpc.dst + tpc.key and the destination later reconnects with
19 * tpc.org + tpc.key; that state is local to the source host.
20 */
21
