6/*
7 * Session lifecycle — the XRootD login sequence every client must complete:
8 *
9 * Client Server
10 * ────── ──────
11 * → kXR_protocol (capabilities) ← kXR_ok (server caps, TLS flags)
12 * → kXR_login (username) ← kXR_ok (session ID) or kXR_authmore
13 * → kXR_auth (credentials) ← kXR_ok or kXR_authmore (multi-round GSI)
14 * → kXR_open / kXR_read / … ← responses
15 * → kXR_endsess ← kXR_ok (connection then closes)
16 *
17 * kXR_protocol must be the first request. kXR_ping is allowed at any time.
18 * All other opcodes require logged_in=1 AND auth_done=1 in brix_ctx_t.
19 *
20 * When brix_auth=none, auth_done is set immediately after login (no kXR_auth
21 * round-trip required). When brix_auth=gsi or token, the client must send
22 * kXR_auth before any file operations.
23 *
24 * GSI uses multiple kXR_auth / kXR_authmore round-trips to exchange DH keys,
25 * certificates, and a signed random challenge. The steps are tracked by the
26 * XrdSutBuffer step numbers in protocol/gsi.h.
27 */
28
