4/*
5 * fs/meta/xmeta_carrier.h — carriers for the unified metadata record (P1).
6 *
7 * WHAT: Persist/load an encoded xmeta record for a store object: preferred
8 * carrier is the reserved "user.xrd.cinfo" xattr (below the VFS seam,
9 * unforgeable by clients — the VFS maps client attrs to user.U.*);
10 * when the store has no xattr surface or the record exceeds the
11 * store's xattr value cap, the identical bytes ride as a co-located
12 * "<key>.cinfo" sidecar object — which, thanks to the stock-compatible
13 * prefix, IS a valid stock XrdPfc cinfo file.
14 *
15 * WHY: Exactly one metadata record per file, wherever the store can keep it
16 * (spec 2026-07-02-xmeta-unified-metadata-design.md §Carriers).
17 *
18 * HOW: Save encodes then TRIES the xattr; E2BIG/ERANGE/ENOSPC/ENOTSUP (or a
19 * missing slot) falls back to a staged sidecar write, then best-effort
20 * removes the xattr so the file never has two carriers. Load checks
21 * the xattr first, then the sidecar. (Try-then-fallback replaces the
22 * spec's startup cap probe: same behavior, no probe machinery, and it
23 * adapts per file rather than per store.)
24 */
