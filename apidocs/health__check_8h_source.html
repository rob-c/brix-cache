1/*
2 * health_check.h — Phase 22 active stream health checks.
3 *
4 * Off by default. When `brix_health_check on` is configured on a
5 * manager/redirector server, each worker runs a timer that periodically claims
6 * a registry slot (exactly one worker probes each data server per interval) and
7 * opens a short-lived XRootD connection to it: TCP connect -> handshake ->
8 * protocol -> login -> kXR_ping (or kXR_stat "/"). A passing probe clears any
9 * health-induced blacklist; repeated failures blacklist the server before
10 * clients are redirected there, catching hung-but-connected nodes that the
11 * passive CMS-disconnect and TCP-keepalive signals miss.
12 */
