1/*
2 * metrics_macros.h — lock-free SHM counter-increment macros and accessor.
3 *
4 * WHAT: Provides the one inline accessor brix_metrics_shared() that resolves
5 * the shared-memory metrics block from ngx_brix_shm_zone, plus the family
6 * of BRIX_*_METRIC_INC / _ADD macros that bump counters for each counter
7 * group: per-server (SRV), WebDAV, S3, and proxy (aggregate + per-upstream).
8 * The low-level BRIX_ATOMIC_INC/DEC/ADD primitives wrap ngx_atomic_fetch_add.
9 * WHY: Metric updates run on the hot request/IO path and across multiple worker
10 * processes, so every counter touch must be lock-free and NULL-safe. Funnelling
11 * all increments through these macros guarantees consistent atomic semantics,
12 * skips work when the SHM zone is not yet initialised (data == NULL or the
13 * nginx "==(void*)1" reuse sentinel) or when a ctx/metrics pointer is NULL,
14 * and keeps call sites terse so handlers never hand-roll atomics.
15 * HOW: brix_metrics_shared() guards the SHM sentinels and returns the typed
16 * block (or NULL). The webdav/s3 macros dereference that singleton; the SRV
17 * and PROXY macros instead take a ctx whose ->metrics field may be NULL. ADD
18 * skips zero/negative amounts. The per-upstream UP_ macros bound-check
19 * proxy_ptr->upstream_idx against BRIX_PROXY_MAX_UPSTREAMS before indexing
20 * the upstreams[] slice, and are meant to be called after the matching
21 * aggregate BRIX_PROXY_METRIC_INC/_ADD. Header-only (static inline + macros);
22 * requires ngx_brix_metrics_t / ngx_brix_shm_zone to be already in scope.
23 */
