4/*
5 * fs_list.h — THE single declaration of every filesystem/storage driver.
6 *
7 * WHAT: the storage-plane sibling of proto_list.h: one X-macro row per
8 * brix_sd_driver_t the module ships, plus the store-URL scheme
9 * alias table. The mechanical enumerations generate from here: the
10 * name→driver registry table (fs/backend/sd_registry.c), the
11 * registry extern declarations (fs/backend/sd.h), and the tier
12 * store scheme table (fs/tier/tier_config.c).
13 * WHY: before this header the census lived in four hand-maintained
14 * places (registry table, sd.h externs, tier_schemes[], and each
15 * driver's .name) — adding a backend was an archaeology exercise,
16 * and nothing tracked WHICH filesystems exist at all.
17 * HOW: X(ID, sym, name, kind)
18 * ID uppercase identity (reserved for future enum surfaces)
19 * sym the symbol fragment: the driver struct is
20 * brix_sd_<sym>_driver in fs/backend/<dir>/sd_<sym>.c
21 * name the driver's .name string — the config backend name,
22 * brix_sd_backend_name() (metric/log label: bounded,
23 * INVARIANT #8), and the tier scheme "driver" column
24 * kind dispatch filter, consumed as BRIX_FS_ROW_<kind>:
25 * BACKEND name-resolvable primary store (registry row +
26 * extern; selected by brix_storage_backend)
27 * ORIGIN remote source driver composed under a cache/
28 * stage tier (created directly, not by name)
29 * DECORATOR wraps another instance (cache/stage/remote)
30 * NEARLINE offline/HSM plane (frm)
31 *
32 * Build-gated rows live in gated sublists so consumers inherit the
33 * #if structure for free. Rows are append-only within their list.
34 *
35 * Adding a filesystem — the full checklist (X = generated from here):
36 * 1. write fs/backend/<dir>/sd_<sym>.c implementing brix_sd_driver_t
37 * (tier-1 rule: ALL raw byte-I/O syscalls live in fs/backend/)
38 * 2. append the row here [X: registry table + census]; declare the
39 * extern + its driver_conf contract in fs/backend/sd.h (hand-written
40 * on purpose — each extern carries the driver's documentation)
41 * 3. add its sources to ./config (+ feature gate if the build must
42 * probe for a library — cf. BRIX_HAVE_CEPH/BRIX_HAVE_SQLITE),
43 * then rm -rf objs && ./configure && make
44 * 4. teach the config-string grammar: bespoke parser branch in
45 * fs/vfs/vfs_backend_config.c (each scheme owns its syntax) and/or
46 * a scheme alias row in BRIX_FS_SCHEME_LIST below [X: tier table]
47 * 5. if tier stores must compose it: creation branch in
48 * fs/tier/tier_build.c (synthesizes the driver conf — semantic,
49 * not generatable)
50 * 6. capability flags honest in sd.h (CAP_FD/SENDFILE/NEARLINE/...) —
51 * the VFS gates paths on them
52 * 7. docs: fs/backend/README.md matrix + CLAUDE.md OP→FILE
53 * 8. tests: data-plane byte-exact PUT/GET + stat/ns + one
54 * security-negative (cf. tests/run_pblock_*.sh, tests/ceph/)
55 */
