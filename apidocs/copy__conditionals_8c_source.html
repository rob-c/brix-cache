4 * WHAT: Evaluates HTTP conditional headers (If-Match / If-None-Match) against the
5 * destination resource during a COPY operation, preventing overwrites when the

8 * WHY: RFC 9110 §13.1.2 defines conditional request semantics for safe operations. COPY

21/*
22 * WHAT: Evaluate HTTP conditional headers (If-Match / If-None-Match) against the COPY
23 * destination resource to enforce optimistic concurrency — prevent overwrites when
24 * client expects specific state or wants to avoid replacing existing content.
25 *
26 * WHY: Without conditionals, a COPY can silently overwrite concurrent modifications by other
27 * clients. RFC 9110 §13.1.2 provides two mechanisms: If-Match (destination must match ETag)
28 * for "overwrite only if unchanged" semantics; If-None-Match (destination must not exist/match)
29 * for "create-only / skip-if-exists" semantics. Both return 412 Precondition Failed on mismatch.
30 *
31 * HOW: First, generate destination ETag from stat metadata (mtime + size) via `webdav_etag_str()`.
32 * Then evaluate each header independently: If-Match requires exact ETag match or "*" wildcard;
33 * If-None-Match passes when destination absent OR ETag doesn't match OR "*" wildcard against
34 * non-existent resource. Both headers checked together — either can pass the precondition.
35 */

37webdav_check_copy_conditionals(ngx_http_request_t *r,
38 const char *dst_path, int dst_exists, const struct stat *dst_sb)
