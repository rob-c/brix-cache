4/*
5 * auth_gate_l1.h — per-worker L1 in front of the SHM auth-result cache.
6 *
7 * WHAT: a tiny lockless direct-mapped cache mapping the 32-byte auth-gate cache
8 * key (already a SHA-256, built in auth_gate.c) to its cached grant/deny
9 * verdict. It sits as the L1 in front of the optional cross-worker SHM cache
10 * (auth_cache.c / `brix_kv_t`): an L1 hit returns the verdict WITHOUT taking
11 * the per-zone `ngx_shmtx` spinlock that `brix_kv_get` holds.
12 *
13 * WHY: the auth gate runs on the GSI/authz hot path. When the SHM auth cache is
14 * enabled, every request — even a cache hit — takes the SHM spinlock, so under
15 * high concurrency (grid-scale GSI pilots/FTS) that single lock serializes auth
16 * decisions across all workers and stalls the event loop. This L1 collapses
17 * the repeated-{op,path,identity} case to an O(1) lockless probe, mirroring the
18 * token L1 (src/token/worker_cache.c).
19 *
20 * SCOPE / CORRECTNESS:
21 * - Used ONLY when the operator enabled `brix_auth_cache` (it is an L1 in
22 * front of that L2, not an independent always-on cache) — so disabling auth
23 * caching still re-evaluates every request, and the L1 inherits the L2 TTL.
24 * - Per-conf, so the auth_level / rules / identity that produced a verdict are
25 * implicitly part of the cache identity.
26 * - Keyed by the caller's 32-byte SHA-256 (no re-hash); a full 32-byte compare
27 * guards against bucket aliasing.
28 * - Lockless: per-worker, only touched from the single-threaded event loop.
29 */
