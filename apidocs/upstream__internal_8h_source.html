2/*
3 * WHAT: Internal upstream redirector state definitions — bootstrap phase enum,
4 * connection state enum, and brix_upstream_s struct layout. This header is
5 * shared across all upstream source files so each function can access the
6 * same type definitions without pulling in the full nginx module headers.
7 *
8 * WHY: Transparent proxy mode requires a multi-phase bootstrap sequence (handshake,
9 * protocol, TLS upgrade, login) followed by request/response cycles. The upstream
10 * context struct tracks wire buffer positions, response accumulators, client
11 * reference, and saved opcode state end-to-end. Splitting internal types into their
12 * own header keeps upstream.h slim as the public API while all implementation files
13 * share one consistent type definition.
14 *
15 * HOW: Three sections in this header:
16 * Bootstrap phase enum (brix_up_bs_t) — ordered phases from handshake through
17 * authmore to done. Each phase maps to a point in the bootstrap byte sequence.
18 * Connection state enum (brix_up_state_t) — high-level states: connecting,
19 * bootstrap, request, async. Used for logging and state machine transitions.
20 * Struct brix_upstream_s — wire buffer accumulators (rhdr/rhdr_pos/resp_*),
21 * write buffer (wbuf/wbuf_len/wbuf_pos), timer event, client reference pointers,
22 * saved opcode fields, authmore counter. Function declarations for internal API.
23 */

82/* Tear down the upstream (frees timer + TCP conn, detaches client ctx) and send a

88/* Serialize the full plaintext bootstrap (12-byte handshake + kXR_protocol +

98/* Relay one accumulated upstream reply (in up->resp_*) back to the client, re-wrapped
99 * with the client's stream ID. Dispatches by status: redirect/ok forward + cleanup +
100 * resume client read; wait schedules a retry timer; waitresp switches to ASYNC; error
101 * maps to a client error frame; unexpected status aborts. May call brix_upstream_abort
102 * (then `up` is dead) on alloc/arm failure or malformed frames. */
103void brix_upstream_forward_response(brix_upstream_t *up);
104
105/* Advance the bootstrap state machine for one accumulated reply (handshake → protocol
106 * → optional TLS → login → optional ztn auth → done). On reaching DONE, calls
107 * brix_upstream_send_request; otherwise resets the response accumulator and re-arms
108 * the read (posting a synthetic event for epoll-ET coalesced replies). Aborts `up` on
109 * any non-ok status or missing TLS/token config required by the server. */

112/* nginx readable-event callback (ev->data is the upstream conn): accumulates one full

118/* kXR_wait retry-timer callback (ev->data is the upstream): re-sends the saved client

123/* nginx writable-event callback (wev->data is the upstream conn): doubles as connect
124 * completion — on CONNECTING it checks SO_ERROR then transitions to BOOTSTRAP. Drains
125 * any remaining wbuf via flush; once fully sent, arms the read side. Cleans up if the
126 * client session is gone; aborts on connect/write timeout or write error. */

129/* Non-blocking write loop draining up->wbuf[wbuf_pos..wbuf_len) to the upstream.
130 * Returns NGX_OK (fully sent, read event armed), NGX_AGAIN (partial — write event
131 * armed, caller re-enters via write handler), or NGX_ERROR (send/event-arm failure;
