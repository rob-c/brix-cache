1/*
2 * store_policy.h — signing_policy table + X509_STORE ex_data binding.
3 *
4 * WHAT: Builds a per-CA-directory table of parsed signing_policy files and
5 * attaches it (with the operator's signing_policy + CRL modes) to an
6 * X509_STORE, so the shared verifier and every store-rebuild path
7 * inherit enforcement from a single place.
8 * WHY: The verifier (brix_gsi_verify_chain) must decide "may this CA sign
9 * this subject?" without threading extra parameters through every
10 * caller. Binding the table to the store lets the decision travel with
11 * the trust material — including the stream CRL-reload rebuild and the
12 * WebDAV build-once path, which do not share a config object.
13 * HOW: ngx-free at its core (uses signing_policy.h + libc); the OpenSSL glue
14 * lives here. Logging is via a caller-supplied callback so no ngx
15 * symbol is required. The store owns the attached table and frees it.
16 */

49/*

109 * Per-certificate WLCG/IGTF conformance policy applied to every cert in a
110 * verified chain: minimum key strength (RSA >= 2048, EC >= 256), no weak
111 * signature algorithm (MD5/SHA-1), a well-formed serial number (positive,
112 * <= 20 octets, RFC 5280 §4.1.2.2), and no embedded control/NUL bytes in the
113 * subject/issuer DN (RFC 5280 §4.1.2.6). Returns 1 when the cert violates
114 * policy, 0 when it is clean. Proxy certificates are exempt from the serial
115 * ceiling (grid proxies routinely use large timestamp-derived serials).
