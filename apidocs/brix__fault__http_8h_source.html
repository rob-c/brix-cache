1/*
2 * brix_fault_http.h — pure HTTP/1.x request-smuggling surgery for
3 * brix-fault-proxy.
4 *
5 * HTTP/1.1 message framing is famously ambiguous when a chain of proxies and
6 * origins disagree about where one request ends and the next begins. This
7 * module rewrites the header block of a message crossing the wire to manufacture
8 * the classic desync primitives an attacker uses to smuggle a second request
9 * past a front-end into a back-end:
10 *
11 * - CL.TE / TE.CL: present BOTH a Content-Length and a Transfer-Encoding:
12 * chunked header, so a front-end honouring one and a back-end the other
13 * disagree on the body boundary.
14 * - duplicate Content-Length with a different value (ambiguous length).
15 * - Transfer-Encoding header-name obfuscation (space-before-colon, leading
16 * tab, underscore) that a lax parser still honours but a strict one drops.
17 * - naked-LF line endings (bare \n instead of \r\n) — another parser
18 * differential.
19 * - inject an arbitrary header, and append a fully-formed smuggled request
20 * after the body.
21 *
22 * Pure functions over caller buffers (no globals, no I/O). Operates on whatever
23 * header block is present in the read buffer; if no CRLFCRLF terminator is in the
24 * buffer the message is passed through untouched (*applied = 0).
25 */
