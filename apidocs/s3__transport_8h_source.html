4/*
5 * s3_transport.h — server-side libcurl HTTP transport for the shared S3 storage
6 * driver (sd_s3), used by the read-through cache's S3 origin.
7 *
8 * WHAT: The module-side implementation of brix_s3_transport_t — one synchronous
9 * libcurl request + response accessors — so sd_s3's SigV4 / Range-GET / HEAD
10 * protocol logic runs over the server's HTTP stack exactly as it runs over
11 * the client's (client/lib/vfs_s3_transport.c) for the native tools.
12 *
13 * WHY: sd_s3 was client-only because src/ had no HTTP-client substrate to inject.
14 * This is that substrate for the server: it lets the cache front an S3 origin
15 * (brix_cache_origin s3://…) by reusing the whole S3 driver, no protocol
16 * code duplicated.
17 *
18 * HOW: Runs ONLY on the blocking cache-fill thread-pool worker (libcurl easy,
19 * synchronous), never the event loop — matching http_transport.c. Stateless
20 * (tctx unused). The Host header is forced to the bare endpoint host (no
21 * port) to match sd_s3's SigV4 canonical host.
22 */
