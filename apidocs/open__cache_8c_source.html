7 * origin configured it serves via per-slice cache files; otherwise it serves a
8 * whole-file cache hit directly, or delegates to a background origin fill on
9 * miss. Returns kXR_NotAuthorized on ACL failure, kXR_ArgInvalid on a bad path. */

24 /* SECURITY (cache transparency): the cache serve path — both the HIT branch
25 * below and the async fill on MISS — MUST run the SAME full three-tier gate
26 * the direct/non-cache path runs (open_request.c, brix_auth_gate at
27 * BRIX_AUTH_READ): authdb (tier 1), VO ACL (tier 2), AND token scope
28 * (tier 3), evaluated against the EXPORT-root namespace path (acl_path), not
29 * the cache-root path. The old VO-ACL-only check let a principal denied by
30 * authdb or token scope be served bytes that another principal's request had
31 * already pulled into the shared, path-only-keyed cache — a cross-user /
32 * cross-group leak. The fill worker performs no auth, so this open-time gate
33 * is the sole enforcement point. It runs BEFORE the stat() below, so a denied
34 * principal never even probes cache residency (no cache-hit timing oracle),
35 * and returns ctx->write_rc exactly as the direct path does. The decision is
36 * L1/L2 identity-cached, so an authorized principal's repeat hits stay cheap. */
37 if (brix_auth_gate(ctx, c, BRIX_OP_OPEN_RD, "OPEN",
38 clean_path, acl_path, conf,
39 BRIX_AUTH_READ, 0) != NGX_OK) {
40 return ctx->write_rc;
41 }
