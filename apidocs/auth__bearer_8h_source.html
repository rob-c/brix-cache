1/*
2 * auth_bearer.h — WLCG bearer-token helpers for the S3 endpoint.
3 *
4 * WHAT: Two small functions that implement WLCG bearer-token handling for the
5 * S3 REST auth gate. They live in their own file to keep auth_sigv4_verify.c
6 * focused on SigV4 (INVARIANT §6: S3 SigV4 and WLCG bearer are mutually
7 * exclusive per request — never blend auth logic).
8 *
9 * WHY: The S3 auth gate (s3_verify_sigv4) needs to detect and validate Bearer
10 * tokens before falling through to SigV4 processing. Keeping this as a
11 * separate compilation unit makes it independently testable and avoids
12 * entangling JWT logic with SigV4 HMAC logic.
13 *
14 * HOW: s3_bearer_present() is a pure predicate — it reads the Authorization
15 * header and returns 1 if it starts with "Bearer " (case-insensitive).
16 * s3_verify_bearer() validates the token via brix_token_validate(), copies
17 * the claims into the request identity via brix_identity_set_token_claims(),
18 * and returns NGX_OK on success or an already-sent 403 XML error response.
19 */
20

42 * On validation failure, an XML S3 AccessDenied response has already been sent

44 *

46 * reject.
47 */
