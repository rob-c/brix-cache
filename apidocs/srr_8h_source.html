4/*
5 * srr.h — WLCG Storage Resource Reporting (SRR) HTTP/JSON endpoint.
6 *
7 * WHAT: Public types + entry points for a small standalone HTTP sub-module that
8 * serves the WLCG "storageservice" JSON document (Storage Resource Reporting,
9 * schema v4.x) at an operator-chosen location. WLCG accounting tooling
10 * (CRIC, the WLCG storage-space accounting harvester, DIRAC occupancy plugins)
11 * pulls this document straight from an HTTP URL to learn a site's total/used
12 * space per share and the protocol endpoints that serve it.
13 *
14 * WHY: This module deliberately does NOT implement the XRootD UDP f-stream /
15 * g-stream monitoring packet protocol. WLCG storage accounting is already
16 * HTTP/JSON-native via SRR, so an HTTP endpoint integrates with the existing
17 * WLCG stack with far less effort (and no extra collector/shoveler) than the
18 * binary UDP monitoring stack would. Transfer/operation counters remain
19 * available on the Prometheus /metrics endpoint (see src/metrics/).
20 *
21 * HOW: A location with `brix_srr on;` binds ngx_http_brix_srr_handler. The
22 * document is built on each request (builder.c): per-share filesystem space
23 * comes from statvfs(2) via brix_fs_usage_stat(); identity/share/endpoint
24 * metadata comes from the brix_srr_* directives. Mirrors the /metrics
25 * module pattern (src/metrics/module.c + handler.c).
26 */
