8/* brix_cache_read_response — read XRootD server response header + body
9 * WHAT: Reads ServerResponseHdr (fixed-size wire header) via recv_exact, extracts status
10 * and dlen fields via ntohs/ntohl, validates dlen against max_body guard, allocates
11 * body buffer (dlen+1 for null termination), reads body payload, null-terminates.
12 * Returns 0 on success with status/dlen/body populated, -1 on any error.
13 *
14 * WHY: Every XRootD request/response cycle requires parsing the fixed-size response header
15 * before reading the variable-length data body. max_body prevents oversized responses
16 * from exhausting thread-pool memory (malloc on each call). The +1 byte for null
17 * termination enables string operations on error messages and stat strings.
18 * All wire fields are big-endian — ntohs/ntohl convert to host byte order. */
