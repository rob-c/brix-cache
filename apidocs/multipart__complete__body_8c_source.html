9/*
10 * multipart_complete_body.c — CompleteMultipartUpload async body handler.
11 *
12 * WHAT: One function: s3_multipart_complete_body_handler() — invoked as an async callback after
13 * ngx_http_read_client_request_body() completes (POST /bucket/key?uploadId=<id>). Responsibilities:
14 * 1. Extract and validate upload_id from query param + hex-format check via mpu_validate_upload_id().
15 * 2. Locate staging directory via s3_get_mpu_dir(), verify it exists with lstat() → NoSuchUpload 404 if missing.
16 * 3. Open a temp file (fs_path.mputmp) confined to root_canon for atomic assembly.
17 * 4. Concatenate part.1 through part.10000 in ascending order from staging dir — skip ENOENT parts, fail on I/O errors.
18 * 5. fstat assembled temp → generate ETag via s3_etag(). Close fd, rename temp→final path atomically.
19 * 6. Best-effort cleanup of staging directory (mpu_rmdir_recursive) — object is already visible if rename succeeded.
20 * 7. Build CompleteMultipartUploadResult XML (Bucket + Key extracted from fs_path + ETag), send via ngx_chain_t → HTTP 200.
21 * The XML body in the request is informational only; concatenation order is deterministic regardless of part arrival order.
22 *
23 * WHY: Multipart uploads accumulate N staged files on disk — without completion they leave orphan parts consuming space.
24 * The ascending-order concatenation (part.1→part.10000) produces a deterministic final object regardless of the order
25 * parts were uploaded, matching AWS S3 semantics. Atomic rename (temp → final path) ensures clients never see a partial
26 * object — either the old object exists or the new one appears instantly. Staging dir cleanup is best-effort because
27 * once rename succeeds the object is visible; failure to remove staging files wastes disk but doesn't affect correctness.
28 * Metric tracking on I/O failures enables monitoring without log noise.
29 *
30 * HOW:
31 * 1. Extract ctx (fs_path from r->ctx, cf from loc_conf), get method_slot for metrics.
32 * 2. Parse uploadId query param → s3_get_query_param() → InvalidArgument 400 if missing/invalid.
33 * 3. Build mpu_dir = fs_path + "/" + upload_id via s3_get_mpu_dir(). lstat(mpu_dir) → NoSuchUpload 404 if ENOENT.
34 * 4. final_tmp = fs_path.mputmp (snprintf). Open WRITE|CREATE|TRUNC, mode 0644 via brix_vfs_open_fd() (thread-safe confined open).
35 * 5. Loop part_num=1..MPU_MAX_PART_NUMBER: snprintf(part_path = mpu_dir/part.%d), open O_RDONLY confined → ENOENT skip,
36 * read/write loop with copy_buf[65536] — write errors close both fds, unlink temp, metric_internal_error, return 500.
37 * 6. fstat(final_fd) for ETag generation. Close fd. brix_rename_confined_canon(temp→fs_path) → rename error: unlink temp, 500.
38 * 7. mpu_rmdir_recursive(mpu_dir) best-effort (WARN on failure, object already committed).
39 * 8. s3_etag(&st, etag) → snprintf XML response with Bucket/Key(derived from strrchr(fs_path,'/')+1)/ETag into xml_buf[512].
40 * ngx_create_temp_buf(xml_len) → copy into buf → build ngx_chain_t → HTTP 200 + content_length_n = xml_len.
41 */
