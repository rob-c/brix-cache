1/*
2 * xrdckverify.c — verify a file on disk against its recorded checksum.
3 *
4 * WHAT: `xrdckverify [--cache|--storage|--auto] [--algo <name>] [-q] <file>`
5 * recomputes a local file's checksum and compares it to the value already
6 * recorded for that file — in a normal storage endpoint's
7 * "user.XrdCks.<alg>" xattr (or "<file>.cks" sidecar), or a proxy cache's
8 * "<file>.cinfo" / "<file>.meta" record.
9 * WHY: A single offline command to detect at-rest corruption of a cached or
10 * stored file, using the same checksum engine the server runs.
11 * HOW: Thin front-end over brix_cks_verify_file() (lib/cks_verify.c).
12 *
13 * exit: 0 ok · 1 mismatch (corruption) · 2 no recorded checksum / unsupported
14 * algorithm · 3 I/O or access error.
15 */
