4/*
5 * dashboard/noop.c — stub implementation of the entire dashboard public API.
6 *
7 * WHAT: Defines the three SHM zone pointers (ngx_brix_dashboard_shm_zone,
8 * _events_shm_zone, _history_shm_zone) and a do-nothing version of every
9 * symbol that transfer_table.c, events.c, history.c and http_tracking.c
10 * normally provide — the slot ops, event log, history sampler and HTTP
11 * request-tracking hooks. Allocators return -1 / NGX_OK, snapshots return
12 * 0, mutators silently discard their arguments.
13 * WHY: The live monitor can be compiled out without touching every call site.
14 * Callers across the stream and HTTP code reference these symbols
15 * unconditionally; building this file *instead of* the real dashboard
16 * sources satisfies the linker and turns all tracking into cheap no-ops.
17 * (It is intentionally NOT listed in the addon `config`, which wires in the
18 * full implementation; noop.c is the drop-in replacement for a
19 * dashboard-disabled build.)
20 * HOW: Each function matches its real prototype exactly, casts every parameter
21 * to (void) to suppress unused-argument warnings, and returns the neutral
22 * value the callers treat as "no tracking active". Keep this file in sync
23 * with dashboard_tracking.h and dashboard.h whenever the API changes.
24 */
