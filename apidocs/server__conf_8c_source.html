221/*
222 * The merge below is split into one helper per configuration area. Each helper
223 * is a verbatim slice of the original linear merge and is invoked in the SAME
224 * order, so every cross-area derivation still sees its inputs already merged
225 * (e.g. writethrough's origin/decision in merge_proxy_net depends on the cache
226 * fields settled in merge_storage; CMS timeout derivation depends on
227 * cms_interval). Helpers that can fail config validation return char*
228 * (NGX_CONF_OK / NGX_CONF_ERROR); the rest return void.
229 */

231/* Identity & crypto: auth scheme + GSI/pwd, XrdAcc engine (+ native-authdb
