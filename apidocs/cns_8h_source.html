4/*
5 * cns.h — minimal Composite Cluster Name Space (§6).
6 *
7 * WHAT: data servers report namespace mutations (file create/closew, unlink, …)
8 * to the manager, which keeps a path→{size,mtime,server} inventory and can
9 * answer kXR_stat for any cluster file locally — without redirecting the
10 * client to the data server that holds it.
11 * WHY: XrdCnsd-style global namespace: a redirector can stat/size any file in the
12 * federation it manages. The existing CMS registry is prefix-based routing
13 * (which server exports which subtree); CNS adds per-file metadata.
14 * HOW: a new raw CMS frame (CMS_RR_CNS) carries one fixed-layout event over the
15 * existing data-server→manager CMS link. The data server emits on closew
16 * (brix_cns_emit); the manager applies into an in-memory inventory
17 * (brix_cns_apply) and looks it up from the stat handler (brix_cns_stat).
18 *
19 * Scope (v1): in-memory, per-worker inventory — correct for a single-worker
20 * manager (the common redirector config). A cross-worker SHM inventory and full
21 * emit coverage (unlink/mkdir/rmdir/mv) are documented follow-ups.
22 */
23
