46/*
47 * brix_xml_escape — escape XML-special characters in a byte buffer.
48 *
49 * WHAT: Scans src[0..len] and writes the escaped result to dst. Special chars
50 * (& < > " ') are replaced with XML entities; control bytes (<0x20, 0x7f)
51 * are encoded as %XX when BRIX_XML_ESCAPE_CONTROL_PERCENT is set.
52 *
53 * WHY: WebDAV lock owners, S3 error messages, and protocol metadata may contain
54 * XML-sensitive characters. Escaping prevents injection into response bodies.
55 *
56 * HOW: Iterate src byte-by-byte; switch on special chars → emit entity strings;
57 * control chars → emit %XX via brix_hex_nibble(); default → copy raw.
58 * Pre-checks size against escaped_len() to avoid overflow. Null-terminates dst.
59 *
60 * Parameters:
61 * src — input bytes to escape
62 * len — length of the input buffer (src may be NULL when len == 0)
63 * flags — combination of BRIX_XML_ESCAPE_* flags (0 = default behaviour)
64 * dst — output buffer for escaped result
65 * dst_size — total size of the output buffer (must accommodate escaped_len + 1)
66 * written — optional pointer to receive the actual byte count written (excl. NUL)
67 *
68 * Returns:
69 * 0 — success; NUL terminator written at dst[written]
70 * -1 — overflow, badarg (dst NULL or dst_size == 0), or src NULL with len != 0

140 * owner — output buffer for extracted owner identity
141 * owner_len — size of the owner output buffer
142 * exclusive — output pointer: set to 1 if exclusive, 0 if shared scope
143 *
144 * Returns:
145 * 0 — success; owner and exclusive populated
146 * -1 — badarg (NULL pointers, zero-length buffers), or XML exceeds max size
