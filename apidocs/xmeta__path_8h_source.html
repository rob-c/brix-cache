4/*
5 * fs/meta/xmeta_path.h — raw-path carrier for the unified metadata record.
6 *
7 * WHAT: Load/save/remove an encoded xmeta record for a LOCAL file by absolute
8 * path: the record rides in the data file's "user.xrd.cinfo" xattr when
9 * it fits, else as the stock-readable "<path>.cinfo" sidecar. Plus the
10 * per-file read-modify-write lock every path-side writer shares.
11 *
12 * WHY: One carrier implementation for every path-side consumer — the cache's
13 * cinfo layer (fs/cache/cinfo.c) and the CSI integrity engine
14 * (fs/backend/csi_*) persist into the SAME record on the SAME file
15 * under the SAME lock. The SD-instance carrier (xmeta_carrier.h) is the
16 * store-object twin of this file.
17 *
18 * HOW: Pure C, ngx-free (standalone-testable): direct getxattr/setxattr and
19 * file I/O — the paths handed here are svc-owned domains (cache trees)
20 * or export data files reached below the VFS seam, never client-mapped
21 * attribute namespaces.
22 */
