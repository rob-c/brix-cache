7/*
8 * loc_cache.h — dynamic file-location cache (Phase-89 W3, phase-61 App C.1).
9 *
10 * WHAT: A shared-memory map path → (host, port) recording which data node
11 * last asserted kYR_have for a file. Filled by the CMS manager side
12 * when a node answers a kYR_state probe (server_recv_frame.c); consulted
13 * by kXR_locate (locate.c) before falling back to the state fan-out.
14 *
15 * WHY: Static registration is prefix-granular — every node exporting "/"
16 * matches every path, so prefix selection cannot know which node HOLDS
17 * a given file. Stock cmsd answers this with an on-demand kYR_state
18 * query cached per path; this table is that cache. It is SHM so a HAVE
19 * ingested on the worker owning the node connection is visible to the
20 * worker serving the client.
21 *
22 * HOW: Open-addressing table (linear probe over a power-of-two slot count),
23 * fnv1a path hash, TTL-lazy eviction: expired slots are treated as free
24 * by both lookup and insert — no sweeper. The table lives in its own
25 * zone allocated via brix_shm_table_alloc (INVARIANT #10: slab header
26 * preserved, mutex bound to the slab pool's recoverable lock word —
27 * never a bare ngx_shmtx_create).
28 */
