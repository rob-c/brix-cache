38/* ---- namespace lookup cache --------------------------------------------- *
39 * A direct-mapped path->meta cache in front of SQLite. It collapses the
40 * redundant per-op lookups the storage path makes — the existence gate's probe
41 * plus the operation's own lookup, stat-after-write, and the shared-parent probe
42 * every sibling create repeats — into a memory hit, skipping both the SQL
43 * compile and the B-tree descent. POSITIVE entries only: an absent path always
44 * falls through to SQL, so a create immediately following a miss stays correct.
45 * Invalidated on every write to a path (put/touch/remove) and fully cleared on
46 * rename (which reparents whole subtrees). A monotonically-bumped generation
47 * makes the fill-after-miss coherent: if any write lands during the SQL window
48 * the store is skipped, so a stale row can never be cached. Worker-local and
49 * mutex-guarded (the catalog is shared across a worker's threads via FULLMUTEX).
50 */
51#define PBLOCK_NSCACHE_BUCKETS 8192u /* power of two */
52#define PBLOCK_NSCACHE_PATHMAX 256
