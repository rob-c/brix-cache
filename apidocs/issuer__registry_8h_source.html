95/*
96 * Validate a bearer token against the multi-issuer registry:
97 * 1. peek iss → select issuer (deny if unknown)
98 * 2. verify signature/exp/nbf with THAT issuer's keys + audiences
99 * 3. enforce base_path / restricted_path against req_path
100 * 4. run the issuer's authorization_strategy ladder (capability now;
101 * group/mapping land in W1b)
102 * On ALLOW returns 0, fills *claims, and sets *out_issuer_bucket to the issuer's
103 * low-cardinality metric id; on DENY returns -1. `op` selects read vs write
104 * scope direction. macaroon_secret/secret_len are forwarded for macaroon
105 * tokens (NULL/0 if unused).
106 */
