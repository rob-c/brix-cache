1/*
2 * xrdgsitest.c — GSI handshake self-test against an XRootD server.
3 *
4 * WHAT: `xrdgsitest [opts] root[s]://host[:port]` — force GSI auth, perform the
5 * full handshake/login, and narrate the result (caps, the auth choice, TLS,
6 * and the proxy/VOMS the client presented). Exit 0 only if GSI authenticated.
7 * WHY: Users debugging "why won't my GSI proxy work?" want a single command that
8 * isolates the credential + handshake from any data transfer — the diagnostic
9 * aid the stock xrdgsitest provides, here libXrdCl/libXrdSec*-free.
10 * HOW: Thin composition: set auth_force="gsi", brix_connect, then brix_explain_conn
11 * (which already narrates auth + invokes the §15.2 credential introspection).
12 *
13 * Clean-room: composes the public libbrix API only.
14 */
