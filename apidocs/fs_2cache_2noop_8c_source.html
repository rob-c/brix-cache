4/*
5 * noop.c — build-time-disabled stub for the read-through / write-through cache.
6 *
7 * WHAT: Provides the complete public symbol set of the cache subsystem
8 * (brix_cache_open_or_fill, the brix_wt_* write-through API, and the
9 * flush thread/event entry points) as inert no-op implementations.
10 *
11 * WHY: The cache is an optional feature. When it is compiled out, the rest of
12 * the module still references these symbols at link time (open/close
13 * handlers call into the write-through hooks unconditionally). This file
14 * satisfies the linker and guarantees graceful degradation rather than a
15 * build break or a runtime crash when caching is absent.
16 *
17 * HOW: Each function discards its arguments via (void) casts and returns the
18 * "do nothing / not available" sentinel for its contract:
19 * - brix_cache_open_or_fill() answers the client with
20 * kXR_Unsupported via brix_send_error() (the cache cannot serve).
21 * - brix_wt_default_decide() and brix_cache_should_writethrough()
22 * return BRIX_WT_DECISION_DENY (never write back to origin).
23 * - brix_wt_config_init_prefixes() passes the prefix list through
24 * unchanged and returns NGX_OK so config parsing still succeeds.
25 * - brix_wt_flush_on_close()/brix_wt_flush_sync_handle() return
26 * NGX_DECLINED so callers fall through to their non-cache path.
27 * - the flush thread/done callbacks are empty.
28 * This file is the build-time counterpart to the real implementations in
29 * open_or_fill.c, writethrough_decision.c, and writethrough_flush.c.
30 */
