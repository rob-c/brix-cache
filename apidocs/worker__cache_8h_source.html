4/*
5 * worker_cache.h — always-on, per-worker L1 bearer-token validation cache.
6 *
7 * WHAT: a tiny lockless direct-mapped cache that maps a token fingerprint to its
8 * already-validated brix_token_claims_t, so a token presented repeatedly is
9 * validated with crypto + JSON parsing at most once per MAX_TTL window per
10 * worker. It sits as the L1 in front of the OPTIONAL cross-worker SHM cache
11 * (token_cache.c / brix_token_cache_*): L1 hit -> no crypto AND no SHM
12 * spinlock; L1 miss -> fall through to L2/full validation, then populate L1.
13 *
14 * WHY: token validation runs inline on the single-threaded nginx event loop
15 * (RSA/ECDSA verify + base64url decode + JSON parse). Under load, doing that
16 * per request starves the loop of time to service other connections' reads,
17 * producing client-side "HTTP ReadTimeout"s. The pre-existing SHM cache is
18 * (a) opt-in (needs an brix_kv_zone) so most deployments had NO caching, and
19 * (b) takes a per-zone spinlock on every lookup/store, so it still contends
20 * under high concurrency. This L1 is always on and lock-free, collapsing the
21 * overwhelmingly common "same client reuses its token for many requests" case
22 * to an O(1) hash probe.
23 *
24 * SECURITY / CORRECTNESS:
25 * - Only SUCCESSFULLY validated claims are ever stored (never failures).
26 * - Each entry is bounded by min(exp, now + MAX_TTL); a token is therefore
27 * re-validated at least every MAX_TTL even if not expired, so revocation /
28 * key rotation / clock changes are picked up within that bound (same policy
29 * as the SHM cache).
30 * - The cache is PER-CONF, so the issuer/audience/key-set/macaroon-secret that
31 * governed a validation are implicitly part of the cache identity — a token
32 * cached under one server/location is never served to another.
33 * - Keyed by SHA-256 of the raw token bytes (collision-resistant); a 32-byte
34 * fingerprint compare guards against bucket aliasing.
35 * - Lockless by construction: the cache is per-worker and only ever touched
36 * from the event loop (never from AIO threads), which is single-threaded.
37 */
