8 * kXR_Qspace (3015) and kXR_QFSinfo (3017): filesystem capacity reports.
9 *
10 * WHAT: Responds to client queries about local filesystem disk capacity using
11 * statvfs(2). Two opcode variants produce different response formats:
12 * Qspace uses "oss.*" key-value pairs; QFSinfo uses the reference server's
13 * compact numeric format used by locate/redirect logic.
14 *
15 * WHY: XRootD clients (xrdcp, xrdfs) need filesystem capacity information to
16 * decide whether a server has sufficient space for writes. Qspace provides
17 * detailed byte-level reports; QFSinfo provides the compact format that
18 * client redirect logic uses to select writable servers in CMS cluster mode.
19 *
20 * HOW: Call brix_fs_usage_stat(conf->common.root.data, &fsu) → on failure return
21 * kXR_IOError with statvfs error message → on success format response per
22 * opcode variant and send via brix_send_ok(). Both log access events at
23 * INFO level. Always returns NGX_OK from the handler (kXR status encoded in
24 * wire response).

93/*
94 * brix_query_fsinfo — handle kXR_QFSinfo (3017) filesystem capacity query.
95
96 * WHAT: Returns writable flag, free space in MB, and utilization percentage in the
97 * compact numeric format used by XRootD client locate/redirect logic.
98 * Format: wVal freeMB util sVal freeMB util — two identical reports for
99 * writable and staging flags so clients see a single uniform filesystem.
100
101 * WHY: kXR_QFSinfo is the response format that XRootD client redirect logic uses to
102 * determine whether a server can accept writes based on disk space. The compact
103 * numeric format matches the reference xrootd server's QFSinfo output exactly.
104 * Used by CMS cluster manager and kXR_locate to select writable servers.
105
106 * HOW: Call brix_fs_usage_stat(root, &fsu) → if fails increment error metric and
107 * send kXR_IOError → on success: compute util=(used_bytes*100)/total_bytes (0 if

109 * if overflow (>31 bits). snprintf resp as "1 %lld %d 1 %lld %d" with writable=1,

111 * increment OK metric, send via brix_send_ok().

115 * c — nginx connection for logging
116 * conf — server configuration containing root filesystem path
