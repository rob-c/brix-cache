11/**
12 * WHAT: Resolve the request URI to a canonicalised filesystem path then perform
13 * stat(2) on the result — combines ngx_http_brix_webdav_resolve_path() and
14 * stat(2) into a single helper for handlers needing both path and metadata.
15 *
16 * Calls the primary path resolver (ngx_http_brix_webdav_resolve_path()) to convert
17 * r->uri into an absolute canonicalised path within root_canon, then immediately calls
18 * stat(2) on that path to populate sb with file metadata. Returns NGX_OK if both steps
19 * succeed; NGX_HTTP_NOT_FOUND (404) if the resolved path doesn't exist (ENOENT); any
20 * other stat error returns NGX_HTTP_INTERNAL_SERVER_ERROR (500). The caller-supplied
21 * path buffer must be at least WEBDAV_MAX_PATH bytes to accommodate the decoded and
22 * canonicalised output. Used by GET, HEAD, PROPFIND handlers that need both the
23 * resolved path and file metadata in a single atomic operation.
24 *
25 * WHY: Many WebDAV methods (GET, HEAD, PROPFIND) require both the resolved path AND
26 * file metadata — calling resolve_path separately then stat would duplicate error
27 * handling logic across multiple handler files. This helper provides an atomic
28 * two-step operation with consistent error mapping. Per AGENTS.md INVARIANT #4: all
29 * wire paths must go through resolve_path() before any filesystem operations — this
30 * function enforces that invariant by embedding the resolution step into stat itself.
31 * Security invariant: resolved path is confined to root_canon, preventing ../ traversal
32 * attacks even if client supplies malicious URI components.
33 *
34 * HOW: Retrieves WebDAV location configuration via ngx_http_get_module_loc_conf(). Calls
35 * ngx_http_brix_webdav_resolve_path() first — if resolution fails (invalid URI, outside
36 * root_canon), returns immediately with that error code. On successful resolution calls
37 * stat(2) on the resolved path — maps ENOENT to 404, all other errors to 500. Returns NGX_OK
38 * only when both resolution and stat succeed. Caller must ensure path buffer has sufficient
39 * capacity (WEBDAV_MAX_PATH). sb pointer may be NULL if caller needs path only without metadata. */
