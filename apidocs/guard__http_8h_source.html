4/*
5 * guard_http.h — HTTP adapter for the pure-C bad-actor guard core.
6 *
7 * WHAT: declares ngx_http_brix_guard_module — the brix_guard* directives,
8 * per-location config (a built guard_ruleset_t + audit-log file), the
9 * ACCESS-phase classify handler (pre-backend bounce) and the LOG-phase
10 * audit handler — plus the guard_request_t builder they share.
11 * WHY: one nginx HTTP module guards both ARC and XrdHttp/WebDAV backends
12 * behind stock proxy_pass; only the `profile` differs. All classification
13 * logic lives in src/net/guard/ (pure C) — this module only normalizes
14 * ngx_http_request_t into guard_request_t and enforces the verdict.
15 * HOW: ACCESS phase runs guard_classify_pre and returns 403/444 on BOUNCE
16 * (backend never touched); LOG phase maps the response status to an
17 * outcome, runs guard_classify_post, and appends one guard_audit_format
18 * line to the configured audit log for fail2ban.
19 *
20 * Directives (all NGX_HTTP_LOC_CONF, inheritable):
21 * brix_guard on|off enable classification
22 * brix_guard_profile <name> "arc" | "xrdhttp" grammar defaults
23 * brix_guard_default_signatures on|off built-in scanner set (default on)
24 * brix_guard_bounce_status 403|444 pre-backend bounce code (default 444)
25 * brix_guard_audit_log <path> fail2ban audit line destination
26 * brix_guard_signature <substr> extra blocklist substring (repeatable)
27 * brix_guard_valid_prefix <prefix> namespace prefix (repeatable)
28 * brix_guard_valid_method <m> [m...] restrict allowed HTTP methods
29 */
