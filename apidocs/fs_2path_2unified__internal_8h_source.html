1/*
2 * unified_internal.h — cross-file declarations for the unified path resolver.
3 *
4 * WHAT: Shared structs and non-static function declarations for the split of
5 * unified.c into three cohesive translation units — unified.c (public
6 * entry + shared logging), unified_build.c (component validation +
7 * candidate assembly), and unified_resolve.c (canonicalisation,
8 * confinement, and missing-tail/parent fallback). Any symbol DEFINED in
9 * one of those files but REFERENCED from another is declared here.
10 *
11 * WHY: The three files were mechanically split from a single 729-line source
12 * with zero behaviour change. Declaring the cross-file seam in one header
13 * keeps the link contract explicit while leaving file-local helpers
14 * static in their owning translation unit.
15 *
16 * HOW: Include-guarded; pulls in the module core (ngx types) and the public
17 * unified.h (brix_path_* types) so it is self-contained.
18 */

25/*
26 * brix_resolve_ctx_t — file-local resolve context.
27 *
28 * WHAT: Bundles the invariant output/logging cluster threaded through every
29 * resolution stage: the log sink, the canonical export root, the caller's
30 * output buffer (resolved/resolvsz), the optional result record, and the
31 * request depth.
32 *
33 * WHY: These same values flow unchanged from the public entry point down
34 * through candidate resolution, missing-tail/parent fallback, and the
35 * final confinement+stat step. Passing them as one context keeps each
36 * static helper at =<5 parameters without adding globals or changing any
37 * value or ordering.
38 *
39 * HOW: Populated once in brix_path_resolve_cstr() and passed by const pointer
40 * to the helpers, which read root_canon/log/depth and write into
41 * resolved/result exactly as the pre-refactor by-value parameters did.
42 */
