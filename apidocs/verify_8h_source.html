4/*
5 * verify.h — checksum-on-fill integrity for the read-through cache.
6 *
7 * WHAT: After a fill has streamed a complete file into its ".part" staging file
8 * (but BEFORE the atomic rename that publishes it), recompute the file's
9 * content checksum locally and compare it to the digest the origin
10 * advertised. A mismatch discards the part so a corrupted transfer never
11 * becomes a served cache entry; a match records the verified digest in the
12 * file's .cinfo (BRIX_CINFO_F_VERIFIED) for durable provenance.
13 *
14 * WHY: The cache previously trusted whatever bytes arrived from the origin. A
15 * truncated, bit-rotted, or man-in-the-middled transfer would be cached
16 * and served to every subsequent client. Verifying against the origin's
17 * own checksum closes that gap and is the integrity half of XrdPfc that
18 * the module lacked. The policy is FAIL-CLOSED, BEST-EFFORT (the default):
19 * verify whenever the origin can supply a digest we can compute; commit
20 * (flagged unverified) when it cannot; never serve a proven-bad file.
21 *
22 * HOW: The transport (origin/transport.h) reports the origin's advertised
23 * algorithm+hex via its checksum() op. This module opens the part file
24 * (O_RDONLY|O_NOFOLLOW), drives the shared checksum kernel
25 * (brix_checksum_hex_name_fd) for the SAME algorithm name, and compares
26 * the two hex strings case-insensitively. It is transport-agnostic: the
27 * xroot:// driver feeds it a kXR_Qcksum reply, the HTTP driver a Digest
28 * header — the verify logic is identical.
29 */

54/*
55 * Verify the staged part file at `part_path` against `origin` under `mode`.
56 *
57 * Returns:
58 * VERIFIED — origin digest present and matched; out_alg / out_hex hold the
59 * canonical algorithm name and computed hex (for the sidecar).
60 * UNVERIFIED — origin offered no usable digest AND mode==BESTEFFORT: the
61 * caller commits the file but records it unverified. (In
62 * mode==REQUIRE this case returns ERROR instead.)
63 * MISMATCH — origin digest present but differs: caller MUST discard the
64 * part. t error is set (kXR_ChkSumErr).
65 * ERROR — could not compute (open/read failure) or REQUIRE with no
66 * digest. t error is set.
67 *
68 * mode==OFF short-circuits to UNVERIFIED without touching the file.
69 * out_alg (>=16 bytes) / out_hex (>=129 bytes) may be NULL if the caller does
70 * not need the computed values. On VERIFIED the caller persists out_alg/out_hex
71 * into the .meta sidecar (brix_cache_meta_t.cks_alg/cks_hex) it already writes.
72 */
