4/*
5 * cache_admit.h — the shared admission filter for the cache subsystem.
6 *
7 * WHAT: One prefix/size/regex matcher that BOTH the read-through cache (what to
8 * fill from origin) and the write-through cache (what to mirror back) call,
9 * so the two halves are configured with the same shape. Returns ADMIT or
10 * DECLINE for a (path, size) pair.
11 *
12 * WHY: The write-through decision engine (writethrough_decision.c) historically
13 * owned the deny/allow-prefix + size + include-regex logic; the read side
14 * had a thinner ad-hoc copy. Lifting the matcher into one unit gives
15 * config/decision parity and a single place to test the precedence rules.
16 *
17 * HOW: deny prefixes win over allow prefixes; a non-empty allow list makes it a
18 * whitelist; a size over the limit DECLINEs unless an include regex matches;
19 * is_new=1 (a not-yet-existing file, size unknown) skips the size cap. NULL
20 * cfg/path → DECLINE (fail-closed). The prefix arrays hold
21 * brix_wt_prefix_entry_t, the element type the config already parses.
22 */
