4/*
5 * meta_advisory.h — shared "advisory" unix-metadata codec for object-store backends.
6 *
7 * WHAT: A tiny, pure (libc-only, no nginx, no sd.h) codec for a reserved metadata
8 * string that carries POSIX mode/uid/gid/mtime across a backend that has no
9 * native concept of them.
10 *
11 * WHY: XrdCeph/RADOS fakes mode and returns ENOTSUP for chmod; S3 has no POSIX
12 * attrs. To let chmod/utime "stick" and round-trip (the approved *advisory*
13 * model), every such backend stores this one string in a reserved slot
14 * (xattr `user.xrd.unixattr` on RADOS; `x-amz-meta-xrd-unixattr` on S3) and
15 * overlays it on stat. The codec is shared so the format is identical across
16 * backends and the client VFS — a mode set over one access method reads back
17 * over another.
18 *
19 * HOW: Format `v1 mode=0640 uid=1000 gid=2000 mtime=<sec> mtime_ns=<ns>`, only the
20 * present fields after the version token. Decode is forward-compatible: it
21 * skips an unknown version and any unknown `key=val` token. Kept dependency-
22 * free so it unit-tests standalone and links into both the module and
23 * libxrdproto. The overlay-onto-`brix_sd_stat_t` happens at the call site
24 * (it owns both the decoded struct and the backend's base stat).
25 */
