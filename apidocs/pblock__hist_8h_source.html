1/*
2 * pblock_hist.h — Phase-83 F11: versioning + trash/undelete for the pblock driver.
3 *
4 * WHAT: Two lifecycle safety-nets built on the F10 refcounted-blob foundation:
5 * versions — with `versions=N`, an overwrite-publish over an existing
6 * file moves the prior object into `versions(path, gen, …)`
7 * (its blob held, not freed), trimmed to the newest N
8 * generations; the old content survives exactly in gen-N.
9 * trash — with `trash=1`, an unlink moves the object into
10 * `trash(trash_id, path, deleted_at, …)` (blob held) instead
11 * of freeing it; `pblock-fsck --gc` purges rows past the TTL,
12 * and undelete pops the most-recent entry back into the
13 * namespace.
14 *
15 * WHY: Gives lifecycle-tooling tests a real undelete/version-listing surface and
16 * makes destructive-op tests self-auditing: after an overwrite the previous
17 * bytes are exactly in gen-1, and after a delete the object is exactly in
18 * the trash — a standing oracle no other feature provides.
19 *
20 * HOW: Every held blob uses the F10 explicit refcount arithmetic: push bumps the
21 * blob (so the caller's subsequent release only decrements — a copy-on-write
22 * transfer of the reference from the live object to the version/trash row),
23 * trim/undelete release it symmetrically. Bump-before-any-release keeps the
24 * refcount >= 1 across the transfer, so blocks are never removed mid-move. A
25 * failed push returns -1 and the caller falls back to the plain release
26 * (the file is simply overwritten/deleted, no history) — fail-open, never a
27 * lost user op. Both features therefore auto-arm F10 refs. Paths are only
28 * ever bound column values (never interpolated), so a hostile name can
29 * neither inject nor traverse. ngx-free (libc + sqlite3); BRIX_HAVE_SQLITE-
30 * gated.
31 *
32 * Requires: pblock_store.h (pblock_state_t), sd_pblock_catalog.h (pblock_meta)
33 * before inclusion.
34 */

55/* Undelete: pop the most-recently-deleted trash entry for `path` back into the

58 * live object already occupies `path`. 0 or -1/errno. */
