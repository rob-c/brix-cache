4/*
5 * gridftp/ev/ftp_ev.h — event-driven GridFTP gateway: shared per-connection
6 * context, state enum, and cross-file declarations.
7 *
8 * WHAT: declares ftp_ev_t (the per-connection state machine), the control-plane
9 * state enum, the buffer geometry constants, and every function exported between
10 * the ev/ translation units (I/O engine, reply framing, path resolution,
11 * dispatcher, command handlers).
12 *
13 * WHY: this tree drives the whole RFC 959 / GFD.020 dialogue as a non-blocking
14 * nginx STREAM module: the connection's read/write events drive a command→reply
15 * state machine, so a single worker multiplexes many sessions rather than pinning
16 * one client per worker on a blocking socket.
17 *
18 * HOW: brix_ftp_ev_handler() is installed as the stream content handler. It
19 * allocates an ftp_ev_t on the connection pool, wires the read/write event
20 * handlers, and returns to the event loop. All subsequent I/O is buffered:
21 * inbound bytes accumulate in ->buf (framed into command lines), replies
22 * accumulate in ->ob (flushed with short-write handling). Commands are processed
23 * lock-step — the next command is only dispatched once the current reply has
24 * fully drained — which bounds outbound buffering to one reply at a time and
25 * mirrors the half-duplex control-channel semantics FTP clients rely on.
26 */
