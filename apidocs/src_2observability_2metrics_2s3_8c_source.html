3/*
4 * WHAT: Prometheus metrics export for the S3-compatible HTTP endpoint.
5 * WHY: The S3 endpoint receives requests from XrdClS3, aws s3 CLI, and other S3-compatible clients.
6 * These counters track request volume, response status classes, authentication outcomes (SigV4 vs anonymous),
7 * data transfer volumes, range handling, PUT body modes, diagnostic events, and ListObjectsV2 pagination stats.
8 * HOW: Five static string tables map slot indices to low-cardinality label strings per INVARIANT #8; one public function
9 * iterates all counters via ngx_atomic_fetch_add(..., 0) for eventually-consistent Prometheus snapshots.
10 * Each metric line includes HELP description and TYPE counter declaration before the actual data line.
11 */

66/* brix_export_s3_metrics — write all S3-endpoint Prometheus metrics (HELP/TYPE +

70 * body, events, + the ListObjectsV2 stats) using the static name tables above. */
