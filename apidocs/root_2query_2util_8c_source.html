6/*
7 * WHAT: File checksum and digest helpers shared by kXR_Qcksum (checksum query) opcodes.
8 * WHY: Clients need to verify file integrity before or after transfer operations. Adler-32 is the default for xrdcp compatibility;
9 * EVP-based digests support md5, sha1, and sha256 via the brix_checksum_alg directive. Two variants exist per algorithm — fd-based
10 * (for open handles during query) and file-based (path lookup + open + compute). INVARIANT: all path strings sanitized with
11 * brix_sanitize_log_string() before logging to prevent binary data corruption in access logs.
12 * HOW: Adler-32 uses 65536-byte read buffer with pread() loop; A/B accumulator modulo 65521 returns (B << 16) | A. EVP digests create
13 * MD_CTX via EVP_MD_CTX_new(), chain Init_ex → Update loop → Final_ex, free context on all exit paths. File variants wrap fd variants
14 * with brix_open_confined() and close(). All error paths return 0xFFFFFFFF (adler32) or 0 (digest).
15 */

58/*
59 * brix_query_digest_fd — EVP digest from open file descriptor.
60
61 * WHAT: Compute any OpenSSL EVP-compatible digest (md5, sha1, sha256) from an
62 * already-open FD. Returns 1 on success with digest bytes written to out[],
63 * or 0 on error (context freed on all exit paths).
64
65 * WHY: Handles kXR_Qcksum variant where client requests a non-Adler-32 checksum
66 * via an existing file handle during read operations. Supports configurable
67 * algorithms set via brix_checksum_alg directive.
68
69 * HOW: Create EVP_MD_CTX → EVP_DigestInit_ex(md, NULL) → pread loop (65536-byte
70 * buffer): EVP_DigestUpdate on each chunk → handle EINTR retries → break on EOF
71 * → EVP_DigestFinal_ex(out, outlen) → free context. Error paths: free context,
72 * log at ERR level with sanitized path, return 0.

74 * Parameters:

76 * path — filesystem path string for error logging
77 * md — EVP_MD pointer selecting algorithm (md5/sha1/sha256)
78 * out — output buffer for digest bytes (must be EVP_MAX_MD_SIZE or larger)
79 * outlen — output pointer for actual digest length in bytes
80 * log — nginx log pointer for I/O error reporting
