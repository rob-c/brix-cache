3/*
4 * WHAT: Prometheus HTTP exporter for transparent XRootD proxy metrics — upstream connect/auth/op counters,
5 * per-upstream breakdowns, and path-based mutation statistics.
6 * WHY: Proxy mode (nginx → root://backend) needs separate counter families from standalone server to distinguish
7 * client-side vs upstream-side behavior. Connect attempts, auth failures, open/read/write/close counts,
8 * relayed bytes, abandoned handles on disconnect, reconnects after idle drops, path ops (rm/mkdir/rmdir/mv),
9 * and absorbed kXR_wait responses all need observability.
10 * HOW: BRIX_EXPORT_PROXY_COUNTER macro generates HELP/TYPE/value lines for each metric family. Iterates server
11 * slots then per-upstream arrays within each slot. Emits two rows per counter — aggregate row (all upstreams
12 * combined) plus individual upstream rows with upstream= label when _um->label[0] is set. Labels are port,
13 * auth, and optionally upstream name — bounded low-cardinality only.
14 */
15
