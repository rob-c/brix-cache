4/*
5 * ssi.h — byte-exact XrdSsi-over-xroot request/response engine (§7, Tier C core).
6 *
7 * WHAT: a client opens an SSI resource ("/.ssi/<service>"), submits a request via
8 * kXR_write whose offset carries an XrdSsiRRInfo{Rxq,reqId,size}, then waits
9 * for the response via kXR_query(infotype=kXR_Qopaqug, body=RRInfo{Rwt}); the
10 * reply is [XrdSsiRRInfoAttn][metadata][data] (XrdSsiTaskReal::GetResp). A
11 * kXR_query body RRInfo{Can} cancels; large/streamed responses are pulled
12 * with kXR_read(offset=RRInfo{Rxq}).
13 * WHY: real interop with libXrdSsi clients, dispatched to compiled-in native
14 * services (no C++ plugin ABI). The wire codec is byte-exact (ssi_rrinfo).
15 * HOW: brix_handle_open binds a virtual handle whose ->ssi points at an
16 * brix_ssi_req_t. write accumulates the request and, once complete, runs
17 * the service through a synchronous responder that fills the response/
18 * metadata buffers. The query-wait builds the reply via ssi_reply. The
19 * read/write/query hooks are clean early-returns, so the normal data path is
20 * unchanged for non-SSI handles.
21 */
