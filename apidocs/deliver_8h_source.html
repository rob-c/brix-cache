4/*
5 * deliver.h — the single async SSI delivery primitive (server push).
6 *
7 * WHAT: push a request's response / alert / error to the client via an unsolicited
8 * kXR_attn frame, addressed to the streamid the client used on the submit it
9 * had acked with kXR_waitresp.
10 * WHY: real libXrdSsi clients expect the server to push a deferred response
11 * (kXR_asynresp) rather than be polled. This is the one place an async
12 * completion reaches the socket.
13 * HOW: runs ONLY in event-loop context (a timer handler or a thread-pool
14 * completion posted back via aio/resume.c), and only after the caller has
15 * resolved a LIVE session through the generation-guarded registry. It
16 * reuses brix_send_attn_asynresp (src/response/async.c).
17 */
