6 *
7 * WHAT: Sends kXR_status (opcode 4007) response for paged write (kXR_pgwrite, opcode 4016) completion. Allocates ServerStatusResponse_pgWrite structure from connection pool via ngx_palloc(). Sets header fields: streamid from ctx->recv.cur_streamid, status=kXR_status in network byte order via htons(), dlen includes body + pgWrite portion size. Body fields: streamID copied from ctx, requestid = kXR_pgwrite - kXR_1stRequest (offset encoding), resptype=0, reserved zeroed via ngx_memzero(). pgWrite portion contains write_offset as big-endian int64 via htobe64(). Calculates CRC32c checksum over body bytes excluding crc32c field itself using brix_crc32c() helper — stores result in network byte order via htonl(). Queues response for wire delivery via brix_queue_response() with total size including pgWrite portion. Per AGENTS.md INVARIANT #1: kXR_pgwrite requires kXR_status(4007) framing + per-page CRC32c — this function implements that invariant.
8 *
9 * WHY: Paged write completion must include offset position and integrity checksum to allow client verification of data delivery — CRC32c ensures the response body was not corrupted during transmission or processing. The kXR_status framing distinguishes pgwrite status from regular opcode responses, enabling clients to parse multi-page write completions correctly. streamid consistency between header and body prevents cross-stream confusion when multiple concurrent writes are active. Thread safety: operates only on local stack variables (rsp) and provided ctx/c connection; no shared state modification during response construction. */

49/*
50 * pgwrite checksum-error (CSE) retransmit frame.
51 *
52 * WHAT: Sends the "accept-then-correct" kXR_status reply when one or more pages
53 * failed CRC32c verification. Unlike brix_send_pgwrite_status (the
54 * no-error 32-byte frame), this appends a ServerResponseBody_pgWrCSE
55 * trailer plus a big-endian vector of the corrupt pages' file offsets, so
56 * the client knows exactly which pages to resend with kXR_pgRetry. The
57 * status is SUCCESS (kXR_status / kXR_FinalResult), not kXR_error — the
58 * data has already been written; the client must correct it.
59 *
60 * WHY: Stock XRootD (XrdXrootdPgwBadCS::boInfo) reports checksum failures via
61 * this in-band list rather than a hard error so a single corrupt page does
62 * not abort a multi-GB transfer. The two CRC32c covers (body crc32c over
63 * streamID..end, cseCRC over dlFirst..bof end) let the client validate the
64 * frame and the list independently.
65 *
66 * HOW: One pool buffer holds [hdr|bdy|pgw|cse|bof[n]]. bdy.dlen = sizeof(cse) +
67 * n*8 (the trailer the client reads after the 32-byte head); hdr.dlen adds
68 * the same to the base 24. dlFirst/dlLast carry the first/last bad pages'
69 * fragment lengths (short for unaligned first/last pages). Falls back to
70 * brix_send_pgwrite_status when n == 0. */
