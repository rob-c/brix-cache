4/*
5 * impersonate.h — per-request UNIX impersonation (phase 40).
6 *
7 * WHAT: Public types + API for mapping an authenticated identity (GSI DN, token
8 * subject, SSS user, krb5 localname) to a concrete local UNIX account
9 * (uid + primary gid + supplementary gids), and the operating-mode constants
10 * for the `brix_impersonation off|single|map` directive.
11 *
12 * WHY: The gateway can optionally execute namespace/open operations with the
13 * credentials of the real local user (so files are owned by, and kernel DAC is
14 * enforced for, that user) instead of the single nginx worker uid. The mapping
15 * layer here is the first, self-contained piece: pure identity -> {uid,gid,gids}
16 * resolution with a grid-mapfile + getpwnam + policy guards (never map to a
17 * reserved/system uid). The privileged broker (broker.c) consumes it.
18 *
19 * HOW: `brix_idmap_resolve()` resolves a principal via (1) an optional
20 * grid-mapfile (DN -> local username), (2) a direct getpwnam() of the
21 * principal, then (3) a squash-to-default or deny policy, with a per-process
22 * TTL cache (modelled on src/acc/groups.c). See docs/refactor/phase-40.
23 *
24 * Off-by-default: with `brix_impersonation off` (the default) NONE of this is
25 * reached — the module behaves exactly as before (single worker uid).
26 */
27
