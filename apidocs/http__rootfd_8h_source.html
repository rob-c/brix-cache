4/*
5 * http_rootfd.h — persistent confinement rootfd for the HTTP protocols.
6 *
7 * WHAT: Opens the O_PATH directory fd (common->rootfd) that WebDAV and S3 file
8 * operations anchor openat2(RESOLVE_BENEATH) on, mirroring the stream
9 * module's per-worker rootfd (src/config/process.c).
10 *
11 * WHY: The kernel-confinement API (src/path/beneath.h) needs a persistent fd on
12 * the export root. The stream side opens it per worker in init_process by
13 * iterating cmcf->servers. HTTP location config (root_canon) is finalised
14 * per-location in merge_loc_conf, so the natural, walk-free place to open it
15 * is right after canonicalisation succeeds.
16 *
17 * HOW: Called from the WebDAV/S3 merge_loc_conf once root_canon is set. Opens
18 * the fd in the master at config time (inherited by every worker via fork —
19 * an O_PATH directory handle is safe to share) and registers a cf->pool
20 * cleanup so the fd is closed when the cycle is torn down on reload/shutdown
21 * — no fd accumulation across reloads. No-op when the protocol is disabled
22 * or no export root is configured (rootfd stays -1).
23 */
