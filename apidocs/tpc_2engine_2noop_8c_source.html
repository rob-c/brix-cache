1/* File: noop.c — Stub native-TPC implementation for builds without TPC
2 * WHAT: Provides empty/refusing stand-ins for the entire native-TPC public API
3 * (brix_tpc_parse_opaque, the key-registry lifecycle —
4 * brix_tpc_key_configure_registry / _generate_key / _register / _validate
5 * / _consume / _remove, the source-policy gate brix_tpc_check_src_policy,
6 * and the pull entry points brix_tpc_prepare_pull / _start_pull /
7 * _launch_pull). This translation unit is compiled in place of the real
8 * SHM key-registry + pull-thread implementation when native TPC is disabled
9 * at build time, so the rest of the module links and runs unchanged.
10 *
11 * WHY: Native TPC pull depends on a cross-process SHM key registry and a
12 * thread-pool data mover (src/tpc/engine/key_registry.c, launch.c, thread.c,
13 * source.c). Builds that omit those features still reference the same
14 * symbols from the stream dispatch and config layers. Supplying inert
15 * definitions here keeps callsites symbol-resolved while guaranteeing that
16 * any actual TPC request is cleanly refused rather than silently misbehaving.
17 *
18 * HOW: Query/lookup helpers return failure or empty results
19 * (brix_tpc_parse_opaque → -1 after zeroing *out; key validate/consume → 0;
20 * generate_key → empty string; register/remove → no-op). The build-time
21 * configure hook brix_tpc_key_configure_registry returns NGX_OK so module
22 * init succeeds. brix_tpc_check_src_policy denies (returns -1) with the
23 * message "native TPC is disabled at build time" copied via ngx_cpystrn.
24 * The pull entry points brix_tpc_prepare_pull / _start_pull send
25 * kXR_Unsupported via brix_send_error; brix_tpc_launch_pull forwards to
26 * brix_tpc_prepare_pull so every wire-level pull attempt is rejected.
27 * */
28
