1/*
2 * pblock_snap.h — F6 snapshots / instant fixture reset for the pblock driver.
3 *
4 * WHAT: A named point-in-time copy of the whole catalog namespace (the `objects`
5 * and `xattrs` rows), with every referenced blob's refcount held up so the
6 * block store is shared, not copied. Take is O(metadata); restore swaps the
7 * live namespace back to a snapshot's rows; drop releases a snapshot's held
8 * references. The block bytes are shared through the F10 `blobs` refcount
9 * table — so snapshots BUILD ON F10 (the `snap=1` opt auto-arms refcounted
10 * blobs; without them a delete between take and restore would physically
11 * remove blocks a snapshot still needs).
12 *
13 * WHY: Millisecond fixture reset. A populated export can be snapshotted, mutated
14 * destructively by a test, and restored byte-identical in one metadata
15 * transaction — no fixture re-upload. Clone-from-snapshot gives per-test
16 * isolated exports over one shared block store.
17 *
18 * HOW: Single-table design (`snapshots` + `snap_objects` + `snap_xattrs`, the
19 * snapshot name a bound column, never an interpolated identifier) so a name
20 * can never inject SQL; the name is additionally validated to a strict
21 * charset. Take/restore/drop run inside BEGIN IMMEDIATE and recompute every
22 * blob's refcount from (live objects + all snapshot copies) so the F10
23 * release path stays exact. Restore refuses (EBUSY) while any regular-file
24 * handle is open on the export (st->open_files), and clears the nscache
25 * (it replaces the whole namespace). Control is driven through a reserved
26 * namespace on the export root — `mkdir /.pblock/snap/<name>` takes,
27 * `mkdir /.pblock/restore/<name>` restores, `rmdir /.pblock/snap/<name>`
28 * drops — intercepted in the driver's mkdir/unlink before any real catalog
29 * work, so it needs no wire-protocol change. Reached only through the inner
30 * (service) mkdir/unlink paths, so snapshot control is service-only.
31 * ngx-free (libc + sqlite3); BRIX_HAVE_SQLITE-gated like the rest of pblock.
32 *
33 * Requires: pblock_store.h (pblock_state_t) before inclusion.
34 */
