4/* ---- Module: Shared TPC Transfer Registry ----
5 *
6 * WHAT: Public interface to the cross-process registry of in-flight third-party
7 * copies. Declares the lifecycle calls (configure / add / update / remove /
8 * find), the snapshot reader, and the progress-emit shim, plus the
9 * flattened brix_tpc_transfer_snapshot_t used to copy a transfer out of
10 * shared memory with its src_url/dst_path inlined as fixed-size char[].
11 *
12 * WHY: Both transports and the dashboard/metrics readers need a single, stable
13 * view of active transfers that survives across nginx worker processes. The
14 * snapshot type deliberately mirrors brix_tpc_transfer_t but with embedded
15 * char buffers so callers can read a consistent copy without holding the
16 * registry lock or pointing into shared memory.
17 *
18 * HOW: brix_tpc_registry_configure() reserves the shared-memory zone at config
19 * time; brix_tpc_registry_add() publishes a transfer and returns its id;
20 * update/remove/find mutate or look up by id; brix_tpc_registry_snapshot()
21 * bulk-copies up to max_transfers entries; brix_tpc_progress_emit() is the
22 * thin progress hook that forwards bytes_done/state to the registry. The
23 * implementation lives in registry.c (and progress.c for the shim).
24 */

49/* Publish a transfer into a free slot. Copies src_url/dst_path into slot-owned

51 * after return) and stamps a fresh id, started_at/updated_at, and a default
52 * PENDING state when state is 0. Returns the assigned non-zero id, or 0 if the
53 * registry is unavailable or full. Takes the registry lock. */

57/* Set bytes_done on the transfer with the given id and refresh updated_at; also

60 * if the registry is unavailable or id is not found. Takes the registry lock. */

64/* Free the slot holding the transfer with the given id (zeroed for reuse).

67 * Takes the registry lock. */

70/* Best-effort, lock-free lookup by id. Returns a pointer directly into the

76/* Copy up to max_transfers in-use slots into caller-provided out[] as flattened
