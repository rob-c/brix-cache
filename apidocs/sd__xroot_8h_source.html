4/*
5 * sd_xroot.h — remote root:// origin storage driver (read + write data path).
6 *
7 * WHAT: A capability-typed SD driver against a REMOTE XRootD (root:// / roots://)
8 * server. Read slots (open/stat/pread/fstat/close) plus, as of Phase 1 of
9 * the writable-remote-backend work, the WRITE DATA PATH (pwrite/ftruncate/
10 * fsync over kXR_write/_truncate/_sync; caps RANGE_READ|RANDOM_WRITE|
11 * TRUNCATE). It is the root:// sibling of the S3 sd_remote driver: the cache
12 * fill drives the read side driver→driver (open the origin file, pread
13 * ranges into the staged-write sink).
14 *
15 * WHY: It folds the root:// origin onto the SAME SD seam as S3, the local cache,
16 * and the export, so storage is origin-agnostic. The write slots are the
17 * foundation for making a remote root:// a first-class writable backend with
18 * optional local staging (see docs/superpowers/specs/2026-06-29-writable-
19 * remote-root-staged-write-design.md); staged-write and namespace (mkdir/
20 * rename) slots are later phases. Not yet registry-selectable as a primary
21 * export backend (Phase 2 wires that) — today it is cache-constructed only.
22 *
23 * HOW: It wraps the in-process XRootD origin wire client (cache/origin_*.c) —
24 * which needs only a server conf + a logical path — behind the SD vtable.
25 * The per-open object holds a live origin connection + open file handle;
26 * pread issues a kXR_read range into a memory sink. AUTH: anonymous login
27 * (the wire client's native mode). Authenticated root:// origins (bearer
28 * token / GSI X.509 proxy) are filled by the cache's native-client
29 * delegation (brix_cache_origin_proxy / _token), not this in-process
30 * driver — a full in-process root:// token/GSI client is a follow-on (that
31 * auth logic lives in libxrdc, which src/ cannot link). Blocking; runs only
32 * on the cache-fill worker thread. Instance/objects are malloc-owned.
33 *
34 * NOTE: This driver intentionally depends on the cache's origin wire client
35 * (cache/cache_internal.h) — it exists to expose that client as an SD
36 * backend, so the dependency is inherent rather than a layering slip.
37 */
