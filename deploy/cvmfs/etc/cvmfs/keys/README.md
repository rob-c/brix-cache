package and are the upstream-published master public keys used to verify
repository whitelists. They are NOT generated by this project.

Each `keys/<domain>/` directory is a key chain: every `*.pub` inside is a
candidate master key and the whitelist verifier accepts a signature from any
of them (that is how upstream rotates keys). When updating, re-vendor the
whole directory from the current upstream package and refresh the fingerprints
above — never hand-edit a key.
