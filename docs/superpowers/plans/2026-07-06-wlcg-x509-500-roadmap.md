# WLCG x509 500-Conformance — P1–P4 Roadmap

> Companion to [P0 infrastructure plan](2026-07-06-wlcg-x509-500-P0-infrastructure.md). P0 delivers the vehicle; this roadmap specifies the fan-out phases precisely enough to execute with parallel agents/workflows. Each phase is independently testable and commits on green.

**Prereq:** P0 complete (brix_store_configure, forge v2 `Clause`/`build_all`, `ConformanceFleet`, C oracle all green).

---

## P1 — The seven test families (≥500 rows, parallel)

Each family is one module `tests/clauses/<family>.py` exporting `CLAUSES: list[Clause]`. Modules are independent (no shared-file contention) — **one agent per family, in parallel**. `tests/clauses/__init__.py` auto-concatenates and asserts unique ids.

### Clause row template (every row follows this)

```python
from clauses._helpers import clause, ns_ca, eec  # shortcuts over ForgeCtx
CLAUSES = [
    clause(
        id="CHN-001", clause="RFC5280 §4.2.1.9",
        title="CA:FALSE intermediate cannot issue certs",
        expected="reject", surface="davs", group="sp_off_crl_off",
        build=lambda ctx: (lambda ca=ns_ca(ctx):
            ctx.cred([eec(ctx, ca, "/DC=t/DC=x/CN=leaf",
                          issuer_is_ca_false=True), ca], leaf_key=True))),
    ),
    ...
]
```

`surface` selects the vehicle: `davs` (fleet, default), `c-oracle` (malformed-DER / pre-wire), `config` (`nginx -t`). `group` selects the pre-stood server. Proxy rows use `surface="c-oracle"` (WebDAV refuses proxies) unless the optional root:// harness lands.

### P1 per-family task cycle (each agent)

1. Write `tests/clauses/<family>.py` with ≥target `Clause` rows.
2. `PYTHONPATH=tests python3 -c "import clauses; print(len([c for c in clauses.ALL_CLAUSES if c.id.startswith('<PREFIX>')]))"` ≥ target.
3. Run the family through the C oracle: `bash tests/c/run_x509_oracle.sh` — record which rows the oracle's verdict ≠ `expected` (these are **candidate gaps** for P2, NOT failures yet; the agent tags them `expected` per **spec-first truth** and lets P2 fix code).
4. Commit `test(x509): <FAMILY>-* clause family (N rows)`.

**Merge step (sequential, after fan-out):** run `tests/test_x509forge_v2_selftest.py` (unique-id + materialization) and confirm `len(ALL_CLAUSES) ≥ 500`.

### P1 completion gate

- `ALL_CLAUSES` ≥ 500, unique ids, every row cites a clause.
- `build_all` materializes cleanly; `manifest.json`/`manifest.tsv` written.
- A pytest driver `tests/test_wlcg_conformance_matrix.py` (added here) parametrizes over the manifest and, for `surface=davs`, asserts `ConformanceFleet.verdict == expected` — **allowed to have failures at end of P1** (those are the gaps P2 closes).

## P2 — Gap closure (fix every gap → zero xfail)

Run the full matrix (oracle + fleet). Every case where our verdict ≠ spec is a gap. **Fix every gap** (spec §1 decision).

### Triage + fix protocol (per gap, adversarially verified)

1. Classify: (a) our enforcement logic missing/wrong, (b) OpenSSL-inherent (fix in our layer — e.g. pre-index CA dir for MD5-only; explicit sig-alg/key-size gate), (c) forge bug (the test itself is wrong → fix the row, not the code).
2. Write/confirm the failing case is spec-correct (cite the clause).
3. Implement the fix in `src/auth/crypto/*` (or a new gate module). Rebuild, `nginx -t`.
4. **Adversarial check:** re-run the WHOLE matrix — the fix must not flip any previously-correct case. If it does, the fix is wrong; revise.
5. Commit per fix or per coherent gap-cluster.

### Anticipated gaps (from clause analysis — not exhaustive)

- **Algorithm policy**: MD5-signed reject, SHA-1 policy gate, RSA<2048 reject, weak-curve reject — likely a new `brix_x509_alg_policy` verify-callback layer (config-gated, secure default).
- **MD5-only CA lookup**: pre-index the CA dir so a CA reachable only by the legacy hash is still found (build an in-memory subject→cert map or add the missing new-hash link at load).
- **Unknown critical extension**: ensure `X509_V_FLAG` / callback rejects unhandled critical extensions (OpenSSL does by default for EEC; verify for proxy/CA).
- **pathLenConstraint** on proxies and CAs enforced end-to-end.
- **DN encoding equivalence** in signing_policy matching (normalize before glob).

### P2 completion gate

- Full matrix (oracle + fleet) **zero failures, zero xfails**.
- Every fix has a passing case AND the adversarial full-matrix re-run stays green.
- Module builds; `nginx -t` clean; first-pass suite still green.

## P3 — Source-level XRootD write-up (parallel)

Doc set under `docs/10-reference/conformance/`. **One agent per component** reads the XRootD v6.1.0 source + our code + the standard and drafts that deep-dive; a synthesis agent writes the overview + master matrix.

### Per-component deep-dive template (`<component>.md`)

For each normative requirement in the component:
1. **Requirement** — quote/cite the standard (RFC/IGTF/EACL clause).
2. **Ours** — behavior + `src/…:line`.
3. **XRootD** — behavior + `/tmp/xrootd-src/src/…:line`.
4. **Verdict** — conformant / stricter / divergent / (gap now fixed) + the pinning `TEST-IDs`.

## P4 — Differential + finalize (sequential)

1. Scale the differential runner to the full manifest: replay every `davs` case against the XRootD conformance fleet (XrdHttp), record ours/xrootd/spec; regenerate `docs/10-reference/conformance/differential-findings.md`.
2. Wire the suite into tiers: C oracle in `--pr` (fast); full fleet matrix `slow`/`nightly`; runner opt-in `TEST_X509_DIFF=1`.
3. Update `docs/09-developer-guide/wlcg-ca-conformance.md` to point at the new doc set + how to run each layer at scale.
4. Final verification: oracle 0 fail; fleet matrix 0 fail/0 xfail; differential runs; build + `nginx -t` clean; first-pass + existing suite green.
5. Update memory index entry.

### P4 completion gate = spec §8 acceptance criteria all met.

## Fan-out orchestration (workflows approved)

- **P1**: one Workflow phase, 7 agents (one per family) in `parallel`, then a merge/gate agent. Each writes only its `tests/clauses/<family>.py`.
- **P3**: one Workflow phase, 7 component agents in `parallel` (read-only over `/tmp/xrootd-src` + `src/`), then a synthesis agent for `README.md`.
- **P2/P4**: sequential (gap fixes interact; finalize is linear) — driven inline with adversarial full-matrix re-runs.
