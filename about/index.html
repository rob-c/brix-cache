About the project

One binary.
Four protocols.
One path to disk.

BriX-Cache is a single nginx module that speaks root:// (XRootD), WebDAV, and S3 off the same POSIX tree — and now terminates a full gsiftp:// GridFTP server besides. Every request enters nginx's event loop, passes one shared core that decides auth and handles caching & metrics, then reaches storage through a VFS where all raw file I/O lives in exactly one place.

What shipped this year.

The project moves in phases, and 2026 has been a heavy one — a fourth protocol, identity that survives the proxy hop, real cluster-management parity, and a write path hardened against a fault-injected network. Each card shows its current validation status.

Landed · native-interop tested

A GridFTP door

A full gsiftp:// GridFTP server: GSI / X.509 control-channel auth, parallel MODE E data streams, encrypted (PROT P) data channels, and server-to-server third-party copy — every transfer bridged onto the same POSIX tree, or delegated onward to a root:// origin as the user.

src/protocols/gridftp/ev/ · tests/test_gridftp_gsiftp_ev.py

Landed · opt-in

Identity survives the hop

Each authenticated request reaches the storage origin under its own identity — a per-user GSI proxy, WLCG token, S3 SigV4 key, or CephX keyring — with opt-in minting of short-lived x509 proxies for bearer-only clients. Stock XRootD collapses every proxied request onto one service credential.

src/fs/backend/cred_mint.c · vfs_staged.c

Landed · wire-tested

Real cmsd cluster parity

BriX-Cache speaks the genuine XrdCms / cmsd cluster-management wire: manager ↔ server selection by real load, dynamic file location, forwarded staging, and multi-replica delete fan-out — drop-in interop with an existing XRootD cmsd mesh, every step flag-gated to legacy behaviour by default.

src/net/cms/ · tests/test_cms_*.py

Landed

Write paths with verified completion

Every write path — WebDAV PUT, S3 PutObject, GridFTP STOR, native kXR_write / pgwrite, and TPC commit legs — gained fail-closed truncation guards plus opt-in end-to-end digest verification, each found and pinned with a userland TCP fault injector that corrupts and truncates the wire on purpose.

tests/resilience/ · docs/07-security/hostile-network-lessons.md

Landed

CVMFS, client and cache

A clean-room native CVMFS client with live-repo signature verification, a FUSE writable overlay over a read-only mount, and stock-style on-demand /cvmfs/<repo> automounting with no autofs kernel module or systemd dependency — works unprivileged, in containers, on WSL2.

client/apps/fs/ · docs/05-operations/cvmfs-automount.md

Code-complete · live lab pending

Ceph / RADOS, natively

RADOS object storage as a first-class backend — striper-aware I/O, per-user CephX credentials, staged writes, and hierarchical directory listing projected over a flat key namespace — reached through the same VFS seam as every other backend.

src/fs/backend/rados/ · tests/run_rados_parity.sh

The codebase

Seven concept buckets under src/.

The source is sorted by concept, not by protocol — so shared behaviour is written once and reused across every front-end. Cross-directory includes are src-rooted.

core/

nginx plumbing — compat, types, config, shm, async io/uring

protocols/

one folder per wire protocol — root, webdav, s3, ssi, srr, dig

auth/

identity & authz — gsi, token, sss, krb5, voms, crypto, impersonate

fs/

the VFS — sole storage truth: confine, backends, cache, tier, xfer

net/

scale-out — cms, manager, upstream, proxy, ratelimit, tap, mirror

observability/

metrics, accesslog, dashboard, pmark

tpc/

third-party copy — cross-plane by nature

What holds it together

Rules the whole system keeps.

Auth before a byte moves

GSI/x509 with request signing, WLCG tokens with per-path scope, SSS, host, password, krb5 — decided once, in one gate.

One storage seam

Raw pread/pwrite/sendfile exist only in src/fs/backend/. Everything above speaks to the VFS.

No worker ever blocks

Disk syscalls run on a thread pool; the worker keeps framing protocol and servicing sockets.

Tested against the reference

Whole test modules run unchanged against BriX-Cache and stock xrootd. Any divergence in status or bytes is a failure.
